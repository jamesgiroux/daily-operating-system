# Account Intelligence Reimagination Set

This revised set is anchored in the **actual account detail chapter structure** and a **real Acme intelligence sample** from the devtools fixture.

Source references:

- `src/pages/AccountDetailEditorial.tsx`
- `src-tauri/src/devtools/mod.rs` (`mock-acme-corp`)

The goal in this second pass is different from the earlier exploration:

- keep the established editorial / magazine / Kinfolk-adjacent reading experience
- preserve the **density of intelligence**
- reimagine the **composition**, not the substance

## Revised Files

- `account-intelligence-ledger-v2.html`
- `account-intelligence-feature-v2.html`
- `account-intelligence-review-v2.html`

## Shared Content Model

All three revised mockups include the same underlying material:

- hero headline and executive assessment
- state of play (`working`, `not working`, `unknowns`)
- relationship health dimensions
- stakeholder insights / The Room
- risks, wins, and open commitments
- value delivered / record / evidence
- company context, source count, and report links

## Direction 1: Ledger v2

**Core idea:** the current account page, but more rigorous and more tactile.

This is the closest to the direction you responded to best. It keeps the magazine document structure and margin grid, but adds denser intelligence handling, explicit evidence bands, and a stronger sense of dossier-level seriousness without turning into a dashboard.

## Direction 2: Feature v2

**Core idea:** a long-form editorial feature story.

This one leans hardest into the magazine reference. It keeps all of the intelligence but arranges it like a beautifully edited feature: wide hero, narrow reading columns, pull quotes, chapter transitions, and inset fact panels.

## Direction 3: Review v2

**Core idea:** an internal review document with editorial pacing.

This direction treats the page like a quarterly review issue. Same content, same warmth, but more structured for operational reading: chapter openers, signal spreads, score breakdowns, and a heavier appendix feel.

## What Changed From The First Pass

- less reduction, more real intelligence
- less novelty for novelty's sake
- closer adherence to ADR-0076 / ADR-0077 / current account detail architecture
- differences are primarily in **layout rhythm, section framing, and hierarchy**
