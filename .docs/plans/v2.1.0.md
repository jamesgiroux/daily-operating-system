# v1.1.0 Version Brief

**Thesis:** Local-first AI. v1.0.0 shipped with Claude Code as the only intelligence provider. v1.1.0 makes the AI backend configurable — Ollama for users who want all intelligence to stay on-device, OpenAI API for users with existing GPT-4o subscriptions. The IntelligenceProvider abstraction (I432) makes this a pure configuration change, not an architecture change. The local-first story that was true for data since v0.1.0 becomes true for AI in v1.1.0.

**Status:** Planning stub — not ready to build.

---

## Scope (tentative)

|----|-------|----------|------|

| I433 | Ollama provider — local LLM support (Llama, Mistral, etc.) | P1 | Backend / Intelligence |
| I434 | OpenAI API provider — GPT-4o support with user-supplied key | P2 | Backend / Intelligence |
| I436 | Workspace file deprecation — DB as sole source of truth | P2 | Backend / Architecture |

---

## Not in scope

- New intelligence features (those ship in other versions)
- Changes to the prompt schema or intelligence output format
- Mobile/iOS (separate initiative)
- Team/collaboration features (separate initiative)

---

## Design principle

Provider switching should be invisible to the intelligence pipeline. `intel_queue.rs` doesn't know whether it's calling Claude Code, Ollama, or OpenAI — it calls `state.intelligence_provider.complete(prompt, tier)` and gets a string back. The quality may differ; the contract is the same.
