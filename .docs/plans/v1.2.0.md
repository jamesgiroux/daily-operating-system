# v1.2.0 — Actions & Success Plans: Closing the Loop

**Thesis:** DailyOS extracts actions from calls, emails, and briefings. It has success plans with objectives and milestones. It computes value delivered in reports. But these three systems are disconnected. Completing an action doesn't advance a milestone. Value delivered is ephemeral. The Intelligence Loop is broken for actions: signals emit but nothing listens. v1.2.0 closes every broken loop. Actions feed health scoring. Milestones advance when work is done. Value persists and compounds. The proposed-action pipeline learns from rejection. The zero-guilt promise is delivered: stale actions age out, progress is visible, delegation is tracked.

**Status:** Not started. Depends on v1.0.2 (actions pipeline fix) and v1.1.0 (lifecycle intelligence — milestone auto-completion from lifecycle events is I628, separate from action-driven milestone advancement in I586).

**Branch model:** Short-lived per-issue branches from `dev`. Issues are largely independent; phasing reflects logical build order, not hard dependencies.

**Origin:** Moved from v1.0.2 (planning reorganization 2026-03-21). Pipeline fix issues (I620-I622) moved to v1.0.2.

## Acceptance Criteria

### Loop Closure
1. **Actions feed health scoring.** Completing actions affects engagement and momentum dimensions. Action stagnation emits health signal.
2. **Value delivered persists.** Stored in `entity_assessment`, survives re-enrichment, user edits preserved.
3. **Action → milestone → objective cascade works.** Full chain verified end-to-end.
4. **Intelligence Loop complete for actions.** Completion signals trigger propagation. Velocity feeds health dimensions. Stagnation surfaces as briefing callout.

### Zero-Guilt
5. **Stale actions age out.** Auto-archive after 30 days inactive. Proposed auto-archive after 7 days. P1/objective-linked exempt.
6. **Progress visible.** Completion momentum surface shows weekly velocity, trend, streak.
7. **Delegation tracked.** Waiting-on view with age, overdue, nudge/complete/reclaim actions.

### Intelligence
8. **Recommended actions generated.** 2-3 per entity from enrichment. No duplicates with pending.
9. **Objectives unified.** AI-extracted matched against user-created. Matches add evidence.
10. **Actions auto-link to objectives.** Fuzzy-match with confirmation. Corrections feed learning.
11. **Rejection learning.** Same-title rejections suppressed. Source fatigue at >70% rejection rate.

### Mock Data
12. **Mock data foundation.** `full` scenario seeds 30+ actions across all states.
