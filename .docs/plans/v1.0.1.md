# v1.0.1 — The Correspondent: Email Intelligence for Customer Success

**Thesis:** DailyOS already treats email as a signal — enriching, scoring, linking to entities, and feeding the Intelligence Loop. But the email surface stops short of the CS workflow: users see intelligence but can't act on it. v1.0.1 closes the gap between "here's what your inbox means" and "here's what you should do about it." No compose, no folders, no email client ambitions — just triage, awareness, and action promotion. Every enhancement feeds back into the Intelligence Loop.

**Status:** Re-scoped after validation. Reply-debt and `repliesNeeded` surfaces were intentionally removed from v1.0.1 and are no longer release blockers.

**Branch model:** Short-lived per-issue branches from `dev`. No umbrella branch needed — these are focused, independent enhancements.

## Fixes Shipped (for CHANGELOG)

### Added

- **Reprocess Transcript** — "Reprocess" button in meeting detail folio bar when a transcript is attached. Clears all extraction data (captures, actions, dynamics, champion health, role changes, commitments) and re-runs the full 3-phase pipeline from scratch. No more stuck transcripts with stale or missing outcomes.

### Changed

- **Briefing toast replaces inline banner** — removed redundant "Refresh in progress" banner from daily briefing. Background work status now shows via global toast with descriptive entity names ("Updating Acme Corp…" instead of "Updating insights…").
- **Transcript timeout doubled** — per-phase extraction timeout 30s → 60s (180s total). 30s was too tight for real Granola transcripts, causing all three phases to time out.

### Fixed

- **Migration 070 captures column bug** — INSERT was selecting `sub_type`/`urgency`/`impact`/`evidence_quote`/`speaker` from old captures table before the rebuild added those columns. Fixed to only copy columns that exist on the source table.
- **Briefing schedule filter** — daily briefing was showing all calendar events including personal/solo blocks (0 invitees). Now correctly filters to meetings with attendees only.

## Acceptance Criteria (Release-Level)

1. **Triage workflow functional**: Every email intel item supports Archive (syncs to Gmail), Open in Gmail (deep link), and Pin (local sort boost). Archive has undo.
2. **Commitments become actions**: Any extracted commitment can be promoted to a tracked Action with entity linkage, due date, and owner. Tracked state persists and is visible.
3. **Silence is surfaced**: Accounts that have gone quiet relative to their historical cadence appear in "GONE QUIET" with dismiss capability. Cadence signals feed engagement dimension.
4. **Email-meeting bridge is visible**: Emails from upcoming meeting attendees show meeting tags. Meeting detail shows linked email correspondence.
5. **Intelligence Loop complete**: Every new surface emits signals (pin, archive, cadence drop, action promotion). All signals feed health scoring dimensions. User dismissals feed Bayesian correction.
6. **Mock data foundation**: `full` scenario seeds 15-20 emails with varied priority, enrichment state, entity linkage, cadence data, pinned states, commitments, and meeting overlap. All remaining frontend sections render correctly from mock data.

## Pass-State Plan

Validation on 2026-03-17 confirmed the release still needs the following before it can pass the remaining acceptance criteria:

1. **Pin behavior + signal emission**: sort pinned emails to the top of their score band and emit the missing pin relevance signal.
2. **Commitment tracking UX**: replace one-click promotion with the inline tracking form, persist tracked state from created actions, and store the correct email-commitment source metadata.
3. **Cadence detection rewrite**: replace the current low-volume heuristic with real cadence-threshold detection and emit/dismiss the intended cadence signals.
4. **Meeting-link source alignment**: derive Correspondent meeting tags from the intended bridge data/window and show linked correspondence summaries on meeting detail.
5. **Mock data parity**: tighten `full` scenario seed data so the remaining in-scope sections validate end-to-end.
