# Phase 0 artifact index — DOS-546 WP Studio spike

## How to use this index

- **Implementing a wave-letter issue?** Find the row whose "Consumed by" lists your issue. Open the file. The acceptance criteria in `02-issues.md` cite specific section names — those sections are binding.
- **Reviewing a PR (L2)?** When an L2 finding cites "doesn't match the contract," resolve which artifact owns the contract via this index, then read the named section. If the section is silent on the question, the finding is a Path-α maintenance item (see CLAUDE.md), not a blocker.
- **Running L0 Prep (`/plan-eng-review`) for a wave-letter issue?** Pull the open-questions column for every Phase 0 artifact your issue consumes. Each open question either gets a Phase 1 decision in the L0 Prep packet or gets explicitly deferred to a named follow-up issue. Silent skipping leaves a known gap in the contract.
- **Considering an artifact amendment?** Open an ADR amendment or a new artifact (`16-...`) rather than silently editing the frozen spec. The artifacts are `spec:ready` precisely so that downstream consumers can cite them by hash.
