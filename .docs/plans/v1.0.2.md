# v1.0.2 — Fix & Reconnect

**Thesis:** v1.0.0 shipped with velocity. v1.0.1 added email intelligence. Before building more features, fix what's broken and reconnect what was disconnected during the rapid build phases. This is coherence work, not feature work. The actions pipeline doesn't surface extracted actions. The briefing expansion panels lost their prep previews. Past meetings fade the wrong content. Transcript output lacks editorial structure. Fix it all, then move forward with confidence.

**Status:** Not started. Depends on v1.0.1.

**Branch model:** Short-lived per-issue branches from `dev`.

---

## Principles

1. **Fix before build.** No new features until existing features work correctly.
2. **Reconnection audit.** Sweep the codebase for built-but-not-wired components — exported but never imported, rendered but missing data, designed but disconnected during redesign passes.
3. **Every fix verifiable.** Each issue has a concrete "before" (broken) and "after" (working) that can be demonstrated in the running app.

## Acceptance Criteria

1. **Actions flow end-to-end.** Transcript processed → actions extracted → proposed actions appear on daily briefing AND actions page → accept → pending → complete → signal emitted. Verified with real transcript in running app.
2. **Briefing expansion panels show prep.** Expand an upcoming meeting on the daily briefing. See: calendar description/context, The Room (attendees), PrepGrid (discuss/watch/wins), "Before this meeting" checklist (max 3 actions), "Read full briefing →" link.
3. **Past meeting outcomes are full brightness.** Open a past meeting with transcript outcomes. Outcomes section is full opacity. Pre-meeting prep content below is faded.
4. **Transcript output is editorially structured.** Post-meeting section reads as a structured report: summary → dynamics → categorized outcomes → commitments → actions. Not a flat list.
5. **Reconnection audit documented.** A sweep produces a list of disconnected components. Quick fixes shipped in this version. Remaining items tracked as new issues for future versions.
6. **Transcript filing routes to all entity types.** Process a 1:1 meeting transcript — markdown filed to person directory. Process a project meeting transcript — filed to project directory. Account routing continues to work as before.
7. **MCP query_entity returns current intelligence.** Call `query_entity` for an enriched account. Intelligence summary comes from DB (`entity_assessment`), not from disk file. Verified by deleting `intelligence.json` and confirming MCP still returns full intelligence.
8. `cargo clippy -- -D warnings` — clean.
9. `cargo test` — all pass.
10. `pnpm tsc --noEmit` — clean.
