# v1.1.1 — Portfolio Surfaces

**Thesis:** v1.1.0 fixed the intelligence foundation and built the report suite. v1.1.1 builds the surfaces the VP opens every morning. The portfolio health page aggregates all accounts into an exception-based editorial view. The account detail page renders the richer intelligence (health bands, Glean-sourced titles, coverage gaps) that v1.1.0 made possible. Editorial, not dashboard. Exception-based, narrative-first.

**Status:** Planning.

**Prerequisite:** v1.1.0 shipped. Health scores non-null for 95%+ accounts. Glean person writeback active. VP Account Review and Portfolio Health Summary reports available.

### I492 — Portfolio Health page

**Delivers:** New `/portfolio` route with editorial magazine aesthetic.

**Files:** New `src/pages/PortfolioPage.tsx`, `src/router.tsx` (add route), new components as needed

**Blocked by:** I499, I502, I491

1. Health heatmap: all accounts color-coded by health score, sized by ARR where available. Uses design system palette (sage for healthy, turmeric for moderate, terracotta for at-risk)
2. Exception list: "3 of 18 accounts need attention" with one-line rationale per account. Only surfaces accounts with declining health, at-risk renewals, or meeting gaps — not all accounts
3. Trend arrows per account from `health_trend.direction` (improving / stable / declining)
4. Renewal timeline: horizontal timeline with upcoming renewals + readiness indicators from I490 data
5. One-click access to generate VP Account Review (I489) and Portfolio Health Summary (I491) reports
6. Clicking an account navigates to account detail page
7. Works with 5 accounts and with 25 accounts — layout scales gracefully
8. Editorial aesthetic: Newsreader headlines, section rules, generous whitespace. NOT a dashboard grid. Think "editor's summary page" at the front of a magazine
9. Nav item added for `/portfolio` route
10. Empty state when no accounts exist guides user to create accounts — role-preset-aware copy

### I493 — Account detail enriched intelligence surface

**Delivers:** Enhanced account detail page leveraging v1.1.0 intelligence improvements.

**Files:** Account detail page files in `src/pages/` and `src/components/`

**Blocked by:** I502, I505

1. **Headline**: health score as confidence band ("72 — Stable") with trend arrow and rationale on hover/expand. Confidence from I499 shown when low ("Low confidence — limited signal history")
2. **State of Play** chapter: `relationship_depth` rendered as structured fields — champion strength, executive access, stakeholder coverage percentage, coverage gaps list
3. **The Room** chapter: stakeholders show title/department from Glean (I505) where available + engagement level (active/warm/cold based on last meeting recency)
4. **New Reports chapter**: consolidated access to all report types available for this account. Each shows: report type name, last generated date (or "Not generated"), staleness indicator, one-click generate button
5. Reports chapter respects availability rules: Renewal Readiness only for accounts with renewal date, Coaching Patterns only when 3+ Monthly Wrapped exist
6. Coverage gaps render visibly in State of Play: "No executive sponsor identified" when role is unassigned
7. Health band uses design system colors — not arbitrary hex values
