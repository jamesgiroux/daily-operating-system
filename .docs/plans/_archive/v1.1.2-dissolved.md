# v1.1.2 — Glean Account Discovery

**Thesis:** The cold-start problem is the biggest barrier to VP adoption. "Go set up all your accounts" is a week of data entry. "Here are your accounts — we already know about them" is a five-minute connection. v1.1.2 adds Glean-powered account discovery that imports accounts from the org's CRM with pre-populated context, and ephemeral queries that let a VP ask "what do we know about X?" without creating a persistent entity. Both eliminate friction and demonstrate immediate value.

**Status:** Planning.

**Prerequisite:** v1.1.0 shipped (Glean signal emission, person writeback active). Glean validation gates for Phase 4 passed — confirmed that Glean can enumerate CRM accounts and return structured metadata.

### I494 — Glean account discovery flow

**Delivers:** "Discover from Glean" on Accounts page. Queries Glean for CRM accounts in user's territory. Shows accounts not yet in DailyOS with context preview. One-click import with Glean context pre-populated. Enrichment starts immediately.

**Files:** `src-tauri/src/context_provider/glean.rs` (discover_accounts method), `src-tauri/src/commands.rs` (new command), `src/pages/AccountsPage.tsx` (discovery UI)

1. With Glean connected, "Discover from Glean" button visible on Accounts page
2. Discovery returns accounts from Glean-indexed CRM not already in DailyOS (matched by name + domain)
3. Each discovered account shows available context preview: name, domain, industry (whatever Glean provides)
4. "Add" creates entity with available context pre-populated (no empty shell)
5. Enrichment begins immediately on add — intelligence populated within one intel_queue cycle
6. Button hidden when Glean not connected — no error state, just absent
7. Results cached with 4-hour TTL to avoid repeated Glean queries
8. Handles large result sets gracefully — pagination or "showing first 50" with search
9. Duplicate detection: account already in DailyOS shows "Already added" badge, not a second add button

---

### I495 — Ephemeral account query

**Delivers:** "Tell me about X" for accounts not in the user's book. Queries Glean, synthesizes a one-time briefing (not persisted). Offers "Add to my accounts" follow-up.

**Files:** New Tauri command `query_ephemeral_account`, new component `src/components/editorial/EphemeralBriefing.tsx`

1. Enter account name not in local DB. System queries Glean. Returns briefing with available org knowledge (whatever Glean provides — could be CRM fields, support tickets, meeting mentions, docs)
2. Briefing is transient — not persisted to DB, no entity created
3. "Add to my accounts" follow-up creates entity and begins persistent enrichment (bridges to I494 flow)
4. Without Glean connected: returns "No external context available — connect Glean in Settings to discover accounts"
5. Briefing renders in editorial style — Newsreader headline, section rules, narrative synthesis
6. Loading state shows editorial generating indicator, not a spinner
7. Accessible via Cmd+K search (when I427 ships) or via Accounts page

## Glean Validation Gates (Must Pass Before Implementation)

These questions must be answered with real Glean queries before implementation begins:

1. Can Glean enumerate accounts from Salesforce? Or only search by name?
2. Can we filter by owner/territory? Or do we get the full org's account list?
3. What account metadata comes back? (name only? ARR? industry? contacts? renewal date?)
4. Rate limits on discovery-type bulk queries?
5. Does Glean's MCP expose a "list CRM accounts" tool, or do we need to construct search queries?

Acceptance criteria will be adjusted based on answers. If Glean cannot enumerate accounts (only search by name), I494 changes from "discover" to "search and add" with a different UX.

### End-to-end smoke test

1. Open Accounts page with Glean connected. "Discover from Glean" visible. Click — accounts from CRM appear that aren't already in DailyOS. Add one. Entity created with context. Intelligence populated within one enrichment cycle.

2. Search for an account name not in DailyOS (via ephemeral query). Briefing renders with available Glean context. Click "Add to my accounts" — entity created, transitions to persistent enrichment.

3. Disconnect Glean. "Discover from Glean" button disappears. Ephemeral query returns "No external context available" message.

4. Re-run discovery. Previously added accounts show "Already added" badge. No duplicates created.
