# v0.14.3 Version Brief

**Thesis:** Google Drive becomes a first-class connector alongside Gmail and Google Calendar. Users can link their Drive workspace and selectively sync documents (product decks, playbooks, case studies, contracts, win/loss analyses) into DailyOS. Documents are embedded and retrieved during entity enrichment — a Product Manager's pitch deck automatically surfaces in account intelligence when relevant, a CSM's playbook informs renewal account context. Watch mode via the Drive Changes API keeps documents in sync without polling overhead.

**Status:** Planning.

## Acceptance Criteria

Each criterion must be verified with real data in the running app after a full `pnpm dev` restart. Compilation is not acceptance.

### I426 — Google Drive Connector

1. **OAuth & Auth** — Uses existing Google OAuth token (reuses Calendar/Gmail auth with Drive scope already present). Users can import Drive documents immediately if authenticated. Verify: Click "Google Drive" button in Inbox; Picker opens without additional consent screen (Drive auth already granted).

2. **Picker UI** — User sees Google Picker modal with folder/document browser. Multi-select enabled. "Confirm Import" button batches selection. Entity picker dropdown to link documents to accounts/projects. "Watch for changes" toggle for continuous sync. Verify: Click "Google Drive" → Picker opens → select files → choose entity → click Confirm → documents appear in `Accounts/{id}/Documents/`.

3. **Document import** — Google Docs/Sheets/Slides are exported as markdown (Docs → plain text, Sheets → CSV, Slides → text outline). Stored locally in entity Documents/ directory. Non-markdown formats (images, videos) are skipped. Verify: Import a Google Doc to Acme account → file appears in `Accounts/acme-corp-123/Documents/my-doc.md`.

4. **Watcher integration** — Files written to entity Documents/ are detected by existing file watcher. Watcher automatically enqueues intel_queue for entity intelligence regeneration. No separate processor needed (integrates with existing watcher pattern). Verify: Import doc → file appears in Documents/ → check app logs for intel_queue enqueue event.

5. **Watch mode** — Drive Changes API polled on 60-minute interval (configurable via settings). New/modified documents in watched sources automatically re-downloaded and updated. Incremental sync via pageToken (O(1) complexity). Verify: Modify document in Drive → wait ≤60 min (or click "Sync Now") → local file updates.

6. **Change detection** — Drive Changes API tracks modifications via pageToken. No duplicate downloads (pageToken advances only on successful sync). Documents marked for re-processing when changed. Verify: Modify Drive doc title/content → local file updates on next poll.

7. **Connector status indicator** — Settings → Connectors → Google Drive shows: enabled toggle, watched source count, last sync timestamp, "Sync Now" button. Errors logged to stderr. Verify: Connector card displays "3 documents · Last synced 2 min ago".

8. **Intelligence enrichment integration** — Files in entity Documents/ are automatically available during entity enrichment via existing content retrieval. Enrichment prompt includes document excerpts when relevant to enrichment context. Verify: Import "Acme Playbook" to Acme account → enrich account → enrichment.json includes playbook context.

9. **Entity linking** — Modal's EntityPicker links documents to account/project at import time. Link stored as `entity_id` in `drive_watched_sources` DB table. Verify: Import doc to Acme account → query `SELECT entity_id FROM drive_watched_sources WHERE google_id = '...'` → returns Acme ID.

10. **`cargo test` passes** — All 1000+ existing tests pass. No regressions on watcher, intelligence, or file handling. New google_drive module tested.

4. **Entity enrichment includes doc**: Import "Acme GTM Strategy" doc to Acme account → enrich Acme account → verify strategy doc content appears in enrichment.json intelligence output.

5. **Settings connector card**: Go to Settings → Connectors → Google Drive → verify shows enabled toggle, document count, last sync time, manual "Sync Now" button.

6. **No regressions**: Run `cargo test` → all 1000+ tests pass. No failures on watcher, intelligence, file handling, or existing connectors.

7. **Console clean**: Open DevTools → no errors related to Google Drive, Picker, or file imports.
