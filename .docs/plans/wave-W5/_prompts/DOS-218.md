## Things to be aware of

- This is the **Read pilot** — the first ability to run on the full W3/W4 substrate. Its job is to prove the pattern, not to add features. Parity with the legacy command on bundle-1 is the bar, not improvement.
- The legacy surface is `get_entity_context_entries` at `src-tauri/src/commands/workspace.rs:1237`, which calls `crate::services::entity_context::get_entries`. There is also `get_entity_context_for_prompt` at `src-tauri/src/intelligence/user_context.rs:89-108` used inside intelligence prompts. Understand both before designing the ability.
- **Read category** per ADR-0102: pure read, no mutations, no publish, no side effects. The ability returns data; it does NOT update the DB, trigger enrichment, or emit signals.
- The ability must produce full **Provenance + SubjectAttribution + `source_asof` + Trust band + composition** per the W5 done-when criteria. These come from W3-B (DOS-211), W3-C (DOS-7), W3-G (DOS-299), and W4-A (DOS-5) respectively. Read those L0 plans in `.docs/plans/wave-W3/` and `.docs/plans/wave-W4/`.
- `source_asof` fields on entity context entries — these are user-written notes, not AI claims. Understand how `ItemSource`/`SourceAttribution` applies to user-authored content vs AI-authored content.
- W3-H (DOS-300) adds `temporal_scope` + `sensitivity` to claims. Entity context entries have their own sensitivity implications (user-authored = higher privacy expectation).
- Parity test: the ability output should be equivalent to what the legacy command returns for bundle-1 test fixtures. Any delta is a regression until intentionally accepted.

## Specific design decisions to record in §3

- Input shape: what uniquely identifies "get entity context" — `(entity_type, entity_id)` pair or a `SubjectRef`? If `SubjectRef`, how does the frontend pass it?
- Output shape: does the ability return `Vec<EntityContextEntry>` wrapped in `AbilityOutput<T>`, or does it reshape entries into a claims-oriented view?
- Source attribution for user-authored content: user-written notes have `Actor::User` as source, not an LLM. How does `SourceAttribution` + `source_asof` apply to user-created rows?
- Sensitivity: are entity context entries `sensitivity = 'internal'` or higher? Per ADR-0125 defaults.
- Trust: user-authored content — what `TrustBand` does it receive? Does user provenance = `LikelyCurrent` by definition?
