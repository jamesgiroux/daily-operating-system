## Things to be aware of

- The release gate command is the **v1.4.0 ship criteria in code**. It must exit zero against bundles 1+5 on the seeded workspace or v1.4.0 does not tag.
- Two invocation modes: **automated** (`cargo run --bin release-gate` or `pnpm release-gate` against seeded bundles, hermetic) and **manual** (developer runs against their real dev workspace and captures evidence artifact — this is the "dogfood" gate).
- "Golden Daily Loop" = the core product loop: entity context populated → meeting brief generated → user corrections accepted → next brief reflects corrections. Bundles 1+5 together exercise this full loop.
- The release gate is a **consumer** of W6-A (bundles), W6-B (bleed detection tests), and W5 pilots (abilities running end-to-end). It aggregates their evidence into one exit code + artifact.
- Evidence artifact format matters — it must be machine-readable (for CI) and human-readable (for the v1.4.0 tag proof bundle). Think JSON with a summary markdown section.
- Depends on: W6-A bundles, W6-B detection, W5 pilots (DOS-218 + DOS-219). All must be merged before this gate can produce a passing run.

## Specific design decisions to record in §3

- Binary vs shell script: `cargo run --bin release-gate` gives Rust-native DB/test access; a shell script wrapping `cargo test` is simpler but can't load the workspace DB for manual mode. Which is right?
- Hermetic vs live DB modes: hermetic mode loads bundle SQL into in-memory SQLite; manual mode opens the real dev workspace DB read-only. How does the gate switch modes?
- Evidence artifact: what does `evidence.json` contain? (suite names, pass/fail per invariant, ability latency p50/p99, bleed detection result, correction-resurrection result, timestamp, schema version, git SHA)
- Gate failure behavior: does the gate print a human-readable summary of what failed and why, or just exit non-zero?
- Mandatory vs tracked suites: bundles 1+5 are blocking; bundles 2-4, 6-8 are tracked/non-blocking per wave plan. How does the gate report tracked failures without blocking?
