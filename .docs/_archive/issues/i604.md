# I604 ‚Äî Report Engine Foundation

**Version:** v1.0.3 Phase 1
**Depends on:** None
**Type:** Feature
**Scope:** Backend (migrations, intelligence/prompts, reports/assembly, db/reports, scheduler) + Frontend (animation primitives)

## Problem

The report engine was built before the Intelligence Loop (Phase 2) landed. Every report re-derives intelligence from raw tables via PTY calls, ignoring `entity_assessment`. This produces:

- Inconsistent narratives (reports may contradict what the account detail page shows)
- Slow generation (PTY calls blocking the user for 15-60 seconds)
- Redundant work (AI re-analyzing data already analyzed during enrichment)
- Dead code from multiple rewrites (SWOT has unreachable parse functions, BoB has `#[allow(dead_code)]` blocks)

Additionally, there are no shared animation primitives for report surfaces.

## Design

### Migration: `069_report_precompute.sql`

Three new columns on `entity_assessment`:
- `best_customer_quote TEXT` ‚Äî verbatim quote selected during enrichment
- `recommended_next_steps_json TEXT` ‚Äî structured `[{action, owner_hint, timeline_hint}]`
- `ebr_cache_json TEXT` ‚Äî pre-generated EBR/QBR customer-facing content

Three new tables:
- `portfolio_assessment` ‚Äî singleton row for cross-entity BoB intelligence
- `user_weekly_summary` ‚Äî one row per week, pre-computed report content
- `user_monthly_summary` ‚Äî one row per month, pre-computed report content

Full DDL in v1.0.3.md schema section.

### Enrichment Prompt Extensions

Extend the existing enrichment prompt schema (small additions, no separate PTY call):

1. **Risk status**: Add `status: "open|mitigated|resolved"` to each risk item in `risks_json`. ~10 tokens per risk.
2. **Structured next steps**: Add `recommendedNextSteps: [{action, ownerHint, timelineHint}]` alongside existing `recommendedActions`. ~100 tokens.
3. **Best customer quote**: Selected deterministically from `captures.evidence_quote` via SQL during enrichment (not AI ‚Äî pure query, most recent positive-sentiment quote).

Update `intelligence/io.rs` to persist these to the new columns.

### Assembly Engine: `reports/assembly.rs`

Pure functions that build report content structs from stored `entity_assessment` data. No PTY calls, no raw table queries. Fully testable.

```rust
pub fn assemble_account_health(intelligence: &IntelligenceJson) -> AccountHealthContent;
pub fn assemble_swot(intelligence: &IntelligenceJson) -> SwotContent;
pub fn assemble_ebr_qbr(intelligence: &IntelligenceJson, ebr_cache: &EbrCacheJson) -> EbrQbrContent;
```

Each function maps pre-computed fields to the report output schema:
- `executive_assessment` ‚Üí `overall_assessment`
- `current_state_json.working` ‚Üí `what_is_working`
- `current_state_json.not_working` ‚Üí `what_is_struggling`
- `risks_json` ‚Üí `risks` (with status mapping)
- `health_json.dimensions` ‚Üí engagement cadence, relationship summary (mechanical)
- `health_json.recommended_actions` ‚Üí `recommended_actions`
- `dimensions_json.expansion_signals` ‚Üí `expansion_signals`
- etc.

### DB Module: `db/reports.rs`

CRUD for the three new tables:
- `get_portfolio_assessment()`, `upsert_portfolio_assessment()`
- `get_weekly_summary(week_start)`, `upsert_weekly_summary()`
- `get_monthly_summary(month_start)`, `upsert_monthly_summary()`

### Dead Code Cleanup

- Remove `gather_swot_input` from `services/reports.rs` (dead)
- Remove `parse_swot_response` from `swot.rs` (unreachable)
- Remove `gather_book_of_business_input` legacy path
- Remove `#[allow(dead_code)]` functions from `book_of_business.rs`
- Fix `ebr_qbr.rs:110` `role_title` ‚Üí `title`
- Remove dead `build_report_preamble` call in `monthly_wrapped.rs`

### Frontend: Animation Primitives

Shared components consumed by all report pages:

- `useSlideReveal` hook ‚Äî IntersectionObserver for scroll-snap slide entry
- `AnimatedCounter` ‚Äî count-up with `tabular-nums` JetBrains Mono, ease-out curve
- `Sparkline` ‚Äî SVG polyline with CSS stroke-dasharray draw animation
- `ProgressRing` ‚Äî SVG circle with animated stroke-dashoffset, color by threshold
- `ProgressBar` ‚Äî horizontal animated fill
- Staggered reveal CSS utility (150ms child delay)
- `report-animations.css` with shared keyframes, `cubic-bezier(0.22, 1, 0.36, 1)` timing
- All respect `prefers-reduced-motion`

3. `assemble_account_health()` produces a valid `AccountHealthContent` from `IntelligenceJson` with zero PTY calls ‚Äî verified via unit test
4. `assemble_swot()` produces a valid `SwotContent` from `IntelligenceJson` with zero PTY calls ‚Äî verified via unit test
5. All dead code removed: `gather_swot_input`, `parse_swot_response`, BoB legacy paths, dead preamble call
6. `ebr_qbr.rs` `role_title` bug fixed
7. All animation primitives render correctly and respect `prefers-reduced-motion`
8. Zero new npm dependencies
9. `cargo test` and `pnpm test` pass
