# I616 — Background task restart wrapper with backoff

**Area:** Backend / Infrastructure
**Priority:** P0
**Version:** v1.0.1
**Source:** v1.0.0 GA engineering review (2026-03-16), Issue 3A

---

## Problem

12 background tasks are spawned as fire-and-forget in `lib.rs`. If any panics, that subsystem silently dies for the rest of the session. No restart, no user notification.

Critical tasks at risk:
- `run_intel_processor` — enrichment stops silently
- `run_meeting_prep_processor` — briefings stop updating
- `run_hygiene_loop` — data quality degrades
- `run_calendar_poller` — schedule goes stale
- `run_email_poller` — inbox intelligence stops

## Design

Use `tokio::task::spawn` (child task) inside the outer spawn to get `JoinError` on panic, which is cleaner than `catch_unwind` for async:

```rust
// src-tauri/src/task_supervisor.rs (new file, ~40 lines)

use std::time::Duration;
use tokio::time::sleep;

const MAX_RESTART_DELAY_SECS: u64 = 60;

pub fn spawn_supervised<F, Fut>(name: &'static str, f: F)
where
    F: Fn() -> Fut + Send + Sync + 'static,
    Fut: std::future::Future<Output = ()> + Send + 'static,
{
    tauri::async_runtime::spawn(async move {
        let mut restart_count: u32 = 0;
        loop {
            let result = tokio::task::spawn(f()).await;
            match result {
                Ok(()) => {
                    // Task exited normally (shouldn't happen for infinite loops)
                    log::warn!("[supervisor] {name} exited normally, restarting");
                }
                Err(e) if e.is_panic() => {
                    restart_count += 1;
                    let delay = (5 * restart_count as u64).min(MAX_RESTART_DELAY_SECS);
                    log::error!(
                        "[supervisor] {name} panicked (restart #{restart_count}): {e:?}, \
                         restarting in {delay}s"
                    );
                    sleep(Duration::from_secs(delay)).await;
                }
                Err(e) => {
                    log::error!("[supervisor] {name} cancelled: {e:?}");
                    break; // Don't restart cancelled tasks
                }
            }
        }
    });
}
```

## Scope

### Files

- **New**: `src-tauri/src/task_supervisor.rs` (~40 lines)
- **Modified**: `src-tauri/src/lib.rs` — replace 12 `tauri::async_runtime::spawn` calls with `spawn_supervised`

### Which tasks get supervised

All 12 long-lived loops:
1. `run_intel_processor` (intel_queue.rs)
2. `run_meeting_prep_processor` (meeting_prep_queue.rs)
3. `run_hygiene_loop` (hygiene.rs)
4. `run_calendar_poller` (google.rs)
5. `run_email_poller` (google.rs)
6. `run_capture_loop` (capture)
7. `run_embedding_processor` (embeddings)
8. Scheduler `.run()`
9. Executor `.run()`
10. Quill poller
11. Granola poller
12. Linear/Drive pollers

**NOT supervised** (initialization-only, not loops):
- `init_db_service` — runs once
- `EmbeddingModel::initialize` — runs once
- `run_startup_sync` — runs once

### Migration pattern in lib.rs

Before:
```rust
let intel_state = state.clone();
let intel_handle = app.handle().clone();
tauri::async_runtime::spawn(async move {
    intel_queue::run_intel_processor(intel_state, intel_handle).await;
});
```

After:
```rust
let intel_state = state.clone();
let intel_handle = app.handle().clone();
task_supervisor::spawn_supervised("IntelProcessor", move || {
    let s = intel_state.clone();
    let h = intel_handle.clone();
    async move { intel_queue::run_intel_processor(s, h).await }
});
```

## Acceptance Criteria

1. Each of the 12 long-lived tasks is wrapped in `spawn_supervised`
2. If a task panics, it restarts after backoff (5s, 10s, 15s... capped at 60s)
3. Log entry includes task name, restart count, and panic message
4. Cancelled tasks (shutdown) do NOT restart
5. `cargo test` passes, `cargo clippy -- -D warnings` clean
6. No changes to task entry point signatures — wrapper is external
