# I562 — Glean `chat` PTY Equivalence Test Suite

**Version:** v1.0.0 Phase 5
**Depends on:** I535 (Glean intelligence provider), I554 (enriched prompt schema)
**Type:** Exploration + validation — test every PTY prompt pattern via Glean `chat`
**Scope:** Backend test harness + validation report. No production UI changes.

---

## Problem

I535 designs the Glean-first enrichment path for entity intelligence (the main `intel_queue.rs` PTY call). But DailyOS has **12+ PTY call sites** across entity enrichment, transcript processing, file enrichment, reports (6 types), briefing narrative, week narrative, and meeting prep enrichment. Each uses different prompt patterns and expects different output schemas.

The I559 spike validated Glean `chat` against the entity enrichment schema (I508 + I554). We haven't validated it against the other PTY call patterns. Before shipping Phase 5, we need to know: which PTY calls can Glean `chat` replace, which can it augment, and which are purely local?

---

## Test Methodology

### For each testable PTY call:

1. **Extract the prompt template** from the Rust source
2. **Adapt it for Glean `chat`** — replace local context injection with "search all available sources"
3. **Call Glean `chat`** with the adapted prompt via `dev_explore_glean_tools` or direct curl
4. **Parse the response** — does it produce valid JSON matching the expected schema?
5. **Compare quality** — is the Glean response richer, equivalent, or thinner than PTY output?
6. **Assess latency** — record p50/p90 for 3+ runs
7. **Document** — go/no-go for replacement vs augmentation vs skip

### Test command

Extend `dev_explore_glean_tools` (or create `dev_test_glean_pty_equivalence`) to run the full test matrix against a real account and produce a structured report.

## Deliverable

Validation report at `.docs/research/glean-pty-equivalence-results.md` containing:

1. Per-call-site results table: schema adherence, data quality, latency, go/no-go
2. Recommended integration pattern per call site (replace / augment context / skip)
3. Schema gaps: any I554 fields that Glean can't produce (and why)
4. Prompt adaptation notes: what changes are needed to make PTY prompts work as Glean `chat` prompts
5. Updated I535 scope if any new replacement opportunities are discovered

---

## Out of Scope

- Implementing the replacements/augmentations (that's I535 and report-specific issues)
- Changing production PTY prompts
- Frontend changes

---

## Acceptance Criteria

1. Validation report exists at `.docs/research/glean-pty-equivalence-results.md`.
2. At least 8 of the 12 PTY call patterns tested against Glean `chat` (skip the 4 purely-local ones).
3. Each test includes: the adapted prompt, the Glean response (or error), schema adherence assessment, data quality comparison to PTY output, and latency.
4. Report-type tests (Account Health, EBR/QBR, SWOT, BoB) include assessment of whether Glean's cross-source data produces richer report sections than local-only context.
5. Go/no-go table for all tested call sites with recommended integration pattern.
6. If any new "replace" opportunities are found (beyond entity enrichment), I535 scope is updated with the specific call site and integration design.

### Mock Data
7. Tests use real Glean data (not mock) — this is a live validation against the Glean instance, same as I559.
