# I608 — Monthly Wrapped v2

**Version:** v1.0.3 Phase 4
**Depends on:** I604 (schema, user_monthly_summary table, animation primitives)
**Type:** Enhancement
**Scope:** Backend (reports/monthly_wrapped, scheduler) + Frontend (monthly-wrapped/ page, slides, styles)

## Problem

Monthly Wrapped runs a single monolithic PTY call dumping 50 meeting summaries + 50 captures + 30 champion health rows + 30 dynamics rows into one prompt (~5-15K tokens). It ignores `entity_assessment` entirely, produces generic personality types (AI-hallucinated archetypes with fabricated rarity percentages), has no achievement system, and renders with flat static slides despite being positioned as a celebration.

With `user_monthly_summary`, the report becomes display-only. Stats are mechanical; editorial fields are pre-generated by a background PTY call at month rollover. The delight layer (archetypes, achievements, animations, per-slide atmospheres) makes this the emotional highlight of the app.

## Design

### Monthly Rollover Task

Fires on first app open after month boundary (or 1st of month if app is running):

**Mechanical computation (instant, no PTY):**
- `total_conversations` — meeting count for the month
- `total_entities_touched` — unique entities from `meeting_entities`
- `total_people_met` — unique attendees from `meeting_attendees`
- `signals_captured` — signal count from `signal_events`
- `top_entity_name` / `top_entity_touches` — most-met entity by meeting count
- `priority_alignment_pct` — entities with meetings that month vs. priority-linked entities
- `top_win` — highest-impact capture with `capture_type = 'win'`
- `top_entity_sparkline_json` — weekly touch counts for top entity
- `month_over_month_json` — deltas vs prior month summary row

**Personality archetype (computed in Rust from behavioral data):**

Six archetypes, scored from meeting dynamics and engagement patterns:

| Archetype | Signal | Detection |
|-----------|--------|-----------|
| The Architect | High priority alignment, planning-heavy | `priorityAlignmentPct >= 80` |
| The Firefighter | High signal volume, reactive patterns | Signal count top quartile, many unique entities/day |
| The Diplomat | Balanced talk/listen, many stakeholders | Talk ratio 40-60%, unique people top quartile |
| The Strategist | Deep focus, long meetings | Unique entities below median, avg duration above |
| The Cultivator | Steady streaks, health trending up | Consecutive weeks contacting priority accounts |
| The Explorer | New contacts, expanding network | New people/entities above median |

Rust scores each archetype from available data, selects highest. The selected type and key evidence are passed to the PTY call, which writes only the description narrative.

**Achievement detection (computed in Rust):**

| Achievement | Detection | Slide |
|-------------|-----------|-------|
| Full House | All Tier 1 entities have ≥1 meeting | Top Account |
| The Turnaround | Any entity health_band red/yellow → green | Top Account |
| Deep Listener | Avg talk ratio < 40% (from `meeting_interaction_dynamics`) | Hidden Pattern |
| Priority Lock | `priorityAlignmentPct >= 80` | Priority |
| Zero Escalations | No entity entered red health_band | Volume |
| Streak: N Weeks | Consecutive weeks contacting all priority entities | Priority |

Achievements stored in `achievements_json`.

**AI computation (one background PTY call, `ModelTier::Extraction`, 60s timeout):**
- `moments_json` — curated highlights from entity assessments active that month
- `hidden_pattern` — behavioral insight from meeting dynamics data
- `personality_json.description` — narrative for Rust-selected archetype (receives archetype name + evidence)
- `carry_forward` — forward-framing from watch items + priorities
- `word_one/two/three` — creative editorial summary

The PTY prompt receives: mechanical stats, archetype selection + evidence, achievement list, entity assessment summaries for active accounts, meeting dynamics aggregates. ~1500-2500 tokens — much smaller than today's 5-15K monolith.

**Storage:** Upsert to `user_monthly_summary`.

### Report Display

Reads from `user_monthly_summary`. Instant render. If no row exists: "Your monthly review will be ready on the 1st" placeholder.

### Frontend: Visual Transformation

**Per-slide background colors** via CSS custom property:

| Slide | Background | Mood |
|-------|-----------|------|
| Splash | cream | Clean open |
| Volume | turmeric 6% | Energy |
| Top Account | sage 6% | Growth |
| Moments | cream | Neutral |
| Hidden Pattern | larkspur 6% | Discovery |
| Personality | turmeric 8% | Identity |
| Priority | sage 6% | Alignment |
| Top Win | cream + confetti | Celebration |
| Carry Forward | cream | Transition |
| Close | sage 12% | Resolution |

**Oversized typography:** Key stats at 72-96px Newsreader. Words at 48-64px.

**Animation integration:**
- Volume: 4 `AnimatedCounter` with 200ms stagger
- Top Account: `AnimatedCounter` + `Sparkline` for weekly engagement
- Priority: `ProgressRing` for alignment percentage
- Top Win: confetti (20-30 CSS spans, design token colors, 2s, fires once)
- Close: three words reveal sequentially (800ms intervals)
- Achievement badges on associated slides (asterisk icon, Newsreader name, DM Sans evidence)
- All slides: `useSlideReveal` + staggered reveal
- Theatrical timing (1.2s) — reflective, celebratory mood

**No "Generating..." state** — instant render from stored data.

## Acceptance Criteria

1. Zero PTY calls when opening Monthly Wrapped — content from `user_monthly_summary`
2. Rollover task computes stats mechanically and generates editorial fields via one PTY call (under 2500 tokens)
3. Personality archetype computed in Rust from behavioral data — AI writes description only
4. Achievements detected accurately from real data (Full House only fires when all Tier 1 touched, etc.)
5. Per-slide backgrounds render with correct color tints
6. Volume slide `AnimatedCounter` counts all 4 stats with 200ms stagger
7. `ProgressRing` on Priority slide animates to alignment percentage
8. Confetti fires on Top Win slide only — 20-30 elements, design token colors, ~2s
9. Three-word close reveals sequentially (800ms intervals)
10. Achievement badges appear on correct slides when earned
11. Month-over-month deltas shown when prior month exists
12. `prefers-reduced-motion` disables all animations
13. Graceful empty state when no summary exists
14. In-place editing preserved — edits stored in `user_monthly_summary`
