# I592 — Auto-Link Actions to Objectives

**Version:** v1.0.2
**Area:** Backend / Actions + Success Plans
**Priority:** P2

## Problem

Actions can be manually linked to objectives in The Work section, but there's no automatic linking when actions are created. A transcript extracts "Send Acme the renewal proposal" as a proposed action, and the account has an active objective "Secure Acme Renewal" — but they're not connected until the user manually links them. This is busywork that the system should handle.

## Design

### Auto-Link on Action Creation

In `services/actions.rs`, after `create_action()` or `accept_proposed_action()`:

1. Get active objectives for the action's account (`get_account_objectives(account_id)`)
2. For each objective, get its milestones
3. Compare action title against:
   - Objective titles (broad match, confidence 0.5)
   - Milestone titles (specific match, confidence 0.7)
4. If best match confidence > 0.6: auto-link to that objective (and milestone if > 0.7)
5. Emit `action_auto_linked` signal with match confidence

### Matching Algorithm

Simple semantic similarity — not embedding-based (too heavy for this frequency):
- Normalize both strings (lowercase, strip punctuation)
- Token overlap (Jaccard similarity on word tokens)
- Key noun matching (extract nouns, compare)
- Account context boost: if action and objective share the same account, boost match score by 0.1

Threshold: 0.6 for objective link, 0.7 for milestone link. These are intentionally conservative — false positives are worse than missed links (user can always manually link).

### User Confirmation

Auto-linked actions show a subtle indicator in The Work section: "Auto-linked" badge. User can:
- **Confirm** — removes the badge, treats as permanent link (positive signal)
- **Unlink** — removes the link, emits correction signal (negative signal for this match pattern)

Confirmed/rejected links feed future matching: if "send proposal" was confirmed as matching "Secure Renewal" for this account, boost similar matches in the future.

### Signal Emission

- `action_auto_linked` (confidence = match score) — action was linked to objective/milestone
- `auto_link_confirmed` (confidence 0.9) — user confirmed the auto-link
- `auto_link_rejected` (confidence 0.3) — user unlinked, feeds Bayesian correction

## Acceptance Criteria

1. New actions are automatically checked against active objectives/milestones for the same account
2. Actions auto-link to objectives when title similarity > 0.6
3. Actions auto-link to specific milestones when title similarity > 0.7
4. Auto-linked actions show "Auto-linked" badge in The Work section
5. User can confirm or unlink auto-links
6. Unlink emits correction signal for Bayesian learning
7. False positive rate is low (< 20% of auto-links need to be corrected)
8. Actions for accounts with no objectives are not processed (no wasted computation)
