# I432 — IntelligenceProvider Abstraction — Multi-LLM Backend Trait

**Status:** Open
**Priority:** P1
**Version:** 1.1.0
**Area:** Backend / Architecture

## Summary

Replaces direct `PtyManager` construction in `intel_queue.rs` with an `IntelligenceProvider` trait. This is a pure refactor — no behavior changes, no new features. The two ADR-0086-compliant call sites in `intel_queue.rs` (lines 534 and 597 per the I376 audit) are migrated to call `state.intelligence_provider.complete(prompt, tier)`. All 13 deliberate-exception sites are left unchanged.

The trait is the enabler for I433 (Ollama) and I434 (OpenAI). Nothing in the enrichment pipeline changes until those issues land. This issue only establishes the abstraction and wires `ClaudeCodeProvider` as the default implementation.

## Acceptance Criteria

1. `IntelligenceProvider` trait exists in `src-tauri/src/intelligence/provider.rs` with the signature:
   ```rust
   async fn complete(&self, prompt: &str, tier: ModelTier) -> Result<String, ProviderError>;
   ```
   The trait is `Send + Sync`. `ProviderError` is a new error type in the same file, with at minimum `Timeout`, `ProviderUnavailable`, and `InvalidResponse` variants.

2. `ClaudeCodeProvider` implements the trait in `src-tauri/src/intelligence/provider.rs`. Its `complete()` implementation wraps the existing `PtyManager::for_tier()` construction and invocation. Behavior is byte-for-byte identical to the current `intel_queue.rs` PTY call.

3. `intel_queue.rs` lines 534 and 597 are migrated: they call `state.intelligence_provider.complete(prompt, tier)` instead of constructing a `PtyManager`. The surrounding error handling, timeout logic, and output parsing are unchanged.

4. `AppState` in `src-tauri/src/state.rs` has a new field:
   ```rust
   pub intelligence_provider: Arc<dyn IntelligenceProvider + Send + Sync>,
   ```
   Initialized to `Arc::new(ClaudeCodeProvider::new(...))` during app startup. The existing `pty`-related fields are NOT removed — they are still used by the 13 deliberate-exception sites.

5. `cargo test` passes. `cargo clippy --manifest-path src-tauri/Cargo.toml --workspace --all-features --lib --bins -- -D warnings` passes. No behavior change is detectable in the running app — enrichment still produces the same `entity_intel` output via Claude Code.

## Dependencies

None. Pure refactor of the two intel_queue PTY construction sites.

Unblocks I433 (OllamaProvider) and I434 (OpenAIProvider) — neither can be built until this abstraction exists.

## Notes
