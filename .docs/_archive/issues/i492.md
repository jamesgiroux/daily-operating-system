# I492 — Portfolio Health Page

**Priority:** P1
**Area:** Frontend / Pages
**Version:** 1.1.1
**Depends on:** I499 (health scoring engine), I491 (Portfolio Health Summary report)

## Problem

There is no aggregate view of all accounts. A VP managing 18 accounts must click into each one individually to understand portfolio health. The daily briefing surfaces exceptions but not the full picture — portfolio shape, renewal timeline, relative health across the book. Without this, the app serves individual contributors but not portfolio owners.

The surface must be editorial, not dashboard. No RAG traffic lights, no pie charts, no dashboard grids. Think editor's summary page at the front of a magazine — narrative-first, exception-based, with the full portfolio visible at a glance.

## Design

### Route and Shell

New `/portfolio` route registered in `src/router.tsx`. Uses `MagazinePageLayout` with turmeric atmosphere (accounts identity). Nav item added to `FloatingNavIsland` between "accounts" and "projects". Folio label: "Portfolio".

### Page Structure

```
1. Hero
   - Newsreader 76px headline: "Your Portfolio" (or personality-driven variant)
   - Hero narrative (italic serif 21px): 1-sentence AI synthesis
     e.g., "3 of 18 accounts need attention — renewals are clustering in Q2"
   - Capacity line (mono 11px): "18 accounts · $4.2M ARR · 3 at risk"

2. Exceptions (ChapterHeading: "Needs Attention")
   - Exception list: ONLY accounts with declining health, at-risk renewals, or meeting gaps
   - Each row: account name (Newsreader 19px link), one-line rationale (DM Sans 15px),
     health pill, trend arrow, ARR in mono
   - Rows are thin divider-separated, not cards
   - If no exceptions: EditorialEmpty "All clear — no accounts need immediate attention"

3. Book View (ChapterHeading: "The Book")
   - Toggle between two views: **Heatmap** (visual health grid) and **Buckets** (strategic categorization)
   - Toggle is minimal — two mono text links, active state underlined. Default: Buckets.

   **Buckets view (default):**
   - Four sections, each a thin-divider-separated list:
     "Growth Focus" (sage accent) — accounts to invest in
     "At Risk, Saveable" (turmeric accent) — accounts to rescue
     "At Risk, Save Unlikely" (terracotta accent) — accounts to escalate or accept
     "Autopilot" (cream/neutral accent) — accounts to monitor, not over-invest
   - Each row: account name (Newsreader 19px link), one-line rationale (DM Sans 13px),
     health score (JetBrains Mono 11px), ARR in mono, trend arrow
   - Bucket assignment uses health score + champion status + engagement trend + expansion signals
     (same logic as I491 bucket classification)
   - Empty buckets: show count "0 accounts" in mono, no empty state noise

   **Heatmap view:**
   - All accounts rendered as tiles, color-coded by health score:
     Sage (#7eaa7b) = healthy, Turmeric (#c9a227) = moderate, Terracotta (#c4654a) = at-risk
   - Tile size proportional to ARR where available (min-size enforced for accounts without ARR)
   - Each tile: account name (DM Sans 13px), health score (JetBrains Mono 11px),
     trend arrow (▲ improving / — stable / ▼ declining)
   - Click navigates to account detail
   - Layout: CSS Grid, wrapping, gap: var(--space-md)
   - With 5 accounts: tiles fill width comfortably
   - With 25 accounts: tiles wrap to multiple rows, remain readable

4. Renewal Timeline (ChapterHeading: "Renewal Horizon")
   - Horizontal timeline showing upcoming renewals across the next 12 months
   - Each renewal: account name, date, readiness indicator from I490 data
   - Implemented as a horizontal scroll row of positioned markers on a month grid
   - Months rendered in JetBrains Mono 10px uppercase
   - Account names in DM Sans 13px, positioned at their renewal date
   - Readiness: sage dot (ready), turmeric dot (preparing), terracotta dot (not started)
   - If no renewal dates: section hidden entirely (not empty state)

5. Reports (ChapterHeading: "Reports")
   - Two report actions with editorial treatment:
     "VP Account Review" and "Portfolio Health Summary"
   - Each: report name (Newsreader 19px), description (DM Sans 13px),
     last generated date (mono 11px) or "Not generated"
   - One-click generate button styled as mono uppercase link

6. Finis (FinisMarker)
```

### Empty State

When no accounts exist, render `EmptyState` component with role-preset-aware copy guiding user to create accounts. Use `getPersonalityCopy("portfolio-empty", personality)`.

### Data Requirements

The page needs:
- All accounts with health scores, ARR, renewal dates, health trends
- Exception filtering logic (declining health OR renewal within 90 days OR no meetings in 30 days)
- Report generation status for VP Account Review and Portfolio Health Summary

Use existing `get_accounts_list` Tauri command for account data. Health trend comes from `health_trend.direction` on account intelligence. No new backend commands needed for the initial implementation — the account list already carries health, ARR, lifecycle, and renewal data.

### Scaling Behavior

- 5 accounts: heatmap tiles are large (200px+), single row. Exception list is short. Timeline is sparse.
- 25 accounts: heatmap wraps to 4-5 rows of smaller tiles (min 120px). Exception list may have 5-8 items. Timeline is dense but scrollable.
- Layout uses CSS Grid with `auto-fill` and `minmax()` to adapt gracefully.

## Acceptance Criteria

1. Buckets view (default): accounts sorted into four strategic buckets (Growth Focus / At Risk Saveable / At Risk Save Unlikely / Autopilot) with one-line rationale per account. Bucket assignment uses health score + champion status + engagement trend + expansion signals.
2. Heatmap view: all accounts color-coded by health score, sized by ARR where available. Uses design system palette (sage for healthy, turmeric for moderate, terracotta for at-risk)
3. Toggle between Buckets and Heatmap views. Minimal toggle design (mono text links, underline active state).
4. Exception list: "3 of 18 accounts need attention" with one-line rationale per account. Only surfaces accounts with declining health, at-risk renewals, or meeting gaps — not all accounts
5. Trend arrows per account from `health_trend.direction` (improving / stable / declining)
6. Renewal timeline: horizontal timeline with upcoming renewals + readiness indicators from I490 data
7. One-click access to generate VP Account Review (I489) and Portfolio Health Summary (I491) reports
8. Clicking an account navigates to account detail page
9. Works with 5 accounts and with 25 accounts — layout scales gracefully
10. Editorial aesthetic: Newsreader headlines, section rules, generous whitespace. NOT a dashboard grid. Think "editor's summary page" at the front of a magazine
11. Nav item added for `/portfolio` route
12. Empty state when no accounts exist guides user to create accounts — role-preset-aware copy

## Out of Scope

- Portfolio-level AI synthesis beyond the hero narrative (future)
