# I590 — Waiting-On as First-Class Status

**Version:** v1.0.2
**Area:** Frontend + Backend / Actions
**Priority:** P1

## Problem

The `waiting_on` field exists in the schema and actions can have `status: "waiting"`, but there's no dedicated UI for managing delegated work. "I asked Sarah to send the proposal 5 days ago" lives as a field on an action card — not as a trackable delegation. For CS users, delegation tracking is a core workflow: you delegate to customers, internal teams, and partners constantly. Waiting-on actions should be a first-class view, not a footnote.

## Design

### Frontend — Delegated View

New filter/view on the Actions page: **"Delegated"** tab alongside Suggested/Pending/Completed.

Delegated view shows actions where `waiting_on IS NOT NULL`, grouped by:
1. **Overdue delegations** (past due_date) — red accent, sorted oldest first
2. **Aging delegations** (no due_date, waiting > 7 days) — amber accent
3. **Active delegations** (recent, on track) — default styling

Each item shows:
- Action title
- "Waiting on: {name}" with waiting duration ("5 days")
- Account name (linked)
- Due date if set
- Actions: **Nudge** (opens Gmail compose to the person), **Complete** (they delivered), **Reclaim** (move back to pending, clear waiting_on)

### Backend — Delegation Intelligence

New query `get_delegation_summary()`:
```rust
struct DelegationSummary {
    total_waiting: usize,
    overdue_waiting: usize,
    avg_wait_days: f64,
    longest_wait: DelegationItem, // most overdue delegation
    by_person: Vec<(String, usize)>, // person name → count waiting on them
}
```

### Integration with Reply Debt (I577)

Cross-reference: if a `waiting_on` person hasn't emailed in N days AND is in reply debt, surface combined signal: "Waiting on Sarah for proposal (5 days) — no email from her since Tuesday." This connects actions and email intelligence.

Query: For each waiting-on action, check if the `waiting_on` person has a corresponding email sender in reply debt. If yes, annotate the delegation with email context.

### Signal Emission

- `delegation_aging` signal when a waiting-on action exceeds 7 days without due_date, or exceeds due_date. Confidence 0.6. Feeds engagement dimension.
- `delegation_resolved` signal when waiting-on is cleared (either completed or reclaimed). Confidence 0.7.

### Meeting Prep Integration

Already partially addressed by I587 (action context in meeting prep). Ensure waiting-on actions for meeting attendees are highlighted: "You're waiting on Sarah (attending today) for the proposal."

## Acceptance Criteria

1. "Delegated" tab on Actions page shows all actions with `waiting_on` set
2. Delegations grouped by overdue/aging/active with appropriate visual treatment
3. Each item shows waiting-on person, duration, account, and due date
4. "Nudge" action opens email compose to the waiting-on person
5. "Complete" marks the action complete with delegation-resolved signal
6. "Reclaim" clears `waiting_on` and moves to pending status
7. Cross-reference with reply debt: if waiting-on person is also in email reply debt, show combined context
8. `delegation_aging` signal emitted at 7-day mark or past due_date
9. Delegation summary available for meeting prep context (I587 integration)
10. **Mock data**: Delegated tab shows 3 items in correct groups (overdue/aging/active). Jamie Rodriguez delegation shows reply debt cross-reference from email data.
