# I280 — Beta Hardening Umbrella — Dependency, DB, Token, DRY Audit

**Status:** Open (Beta Gate)
**Priority:** P1
**Version:** 1.0.0 (Beta Gate prerequisite)
**Area:** Code Quality

## Summary

A comprehensive pre-beta audit of the codebase covering four dimensions: (1) dependency and bundle health — unused npm packages, oversized dependencies, lockfile hygiene; (2) database schema integrity — missing foreign key enforcement, index coverage, schema consistency; (3) token and prompt efficiency — identifying high-cost AI calls that can be reduced without quality loss; (4) DRY debt — duplicate code patterns formalized and extracted.

This umbrella issue was shipped partially in v0.8.0: the audit itself was done (I280) and 4 of the 12 sub-issues (I281-I284) were resolved. The remaining sub-issues (I285-I291) cover ongoing hardening work not yet completed.

## Acceptance Criteria

Not yet specified as a single blocking list. The beta gate requires a 30-day stability window with no open P0 issues, which implies all critical hardening work is done. The remaining sub-issues need to be evaluated and either closed or explicitly scoped into the 1.0.0 brief.

## Dependencies

- Must be substantially complete before 1.0.0 Beta Gate alongside I056, I057.
- Sub-issues I281-I284 closed in v0.8.0. Sub-issues I285-I291 status varies.

## Notes / Rationale

The original audit (v0.8.0) found: 3 unused npm packages (removed), 13% FK enforcement in DB, ~60% token reduction possible, and known DRY debt. The P1 priority reflects that unresolved hardening issues are technical debt that compounds as features are added. The beta gate requires this to be substantially resolved.
