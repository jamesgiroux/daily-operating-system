## Findings — no Phase 0 drift, three path-α observations

1. **W5-C fixture-bundle cardinality mismatch.** Issue body lists bundles `19-24` (six new bundles); project DoD §12 and wave-plan release gate say `19-23` (five new). Reconcile to one number before W5 kickoff. **Not blocking** — file under maintenance.

2. **W2-A `Origin` guard wording.** Acceptance states "reject any non-empty `Origin` with `http`/`https` scheme; reject `Origin: null`." Strictly correct (legitimate loopback PHP-to-Rust has no `Origin`), but the inverted phrasing invites implementation drift. Recommend reformulating as positive allowlist ("accept only requests with empty/absent Origin OR Origin matching the paired site_url") at W2-A L0 plan. **Not blocking.**

3. **HMAC freshness window not numerically bound in W2-B body.** Issue cites artifact 08 for the seconds. Acceptable (artifact is the spec), but L2 reviewers should verify artifact 08 carries an explicit number (e.g., 300s) and that the bound is centrally configurable, not float-literal'd in code.
