## Findings

### 1. MCP actor projection is wrong (BLOCKING — AC line 546)

`src-tauri/abilities-runtime/src/inventory.rs:284` maps `ActorKind::Agent | System | Admin` to the `runtime` actor in the projected inventory. The Phase 0 artifact 05 schema declares `mcp_client` as a distinct actor (line 46) and requires it when `mcp_exposure = "invocable"` (line 122). As written, the generated inventory can never emit `mcp_client`, so the surface-facing projection contract is broken before W3-C filtering even exists. This is a literal AC violation (line 546 names `actor` as a required field that conforms to artifact 05's enum), not a path-α theoretical hardening.

### 2. `emit_ability_inventory` exit codes reversed from documented contract

Per commit message and review prompt: exit `0` success / `1` registry-violation refusal / `2` I/O error. Actual:
- `src-tauri/abilities-runtime/src/bin/emit_ability_inventory.rs:75` returns `2` for registry violations.
- Lines `85` and `93` return `1` for serialization / write errors.

Codex verified via probe: I/O error returns `exit=1`. CI relies on a stable contract for hint surfacing; flip the values to match the brief, or amend the brief and the commit message. The CI gate currently doesn't depend on the distinction (it shells out to `diff -u` on a tempfile), so this is fix-the-binary, not fix-CI.

### 3. Schema mirror loose vs artifact 05 (minor — AC line 539 fidelity)

Two spots where Rust/TS shapes admit values artifact 05's TS interface would reject:
- `AnnotationValue` in `inventory.rs:192` only accepts integer numbers in Rust, while artifact 05 and the TS mirror allow JSON `number` (float-valid). The Rust side is stricter than the contract, which means a future ability with a float annotation would fail Rust serialization while passing the TS check — projection asymmetry.
- `CompositionKind` is loosened from artifact 05's discriminated union into a `boolean + array` shape in both Rust (`inventory.rs`) and TS (`web/types/ability-surface.ts:62`). Invalid combination `{ produces_blocks: false, block_types: [...] }` is type-valid. Artifact 05's union enforces "if `produces_blocks: false`, no `block_types`."

Acceptance line 532 says "TypeScript mirror mirrors the Rust shape verbatim" — both files match each other but neither matches artifact 05's discriminated union exactly. Treatable as a single class fix: tighten both sides to the artifact's TS interface.
