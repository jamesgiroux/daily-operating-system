## New Findings (introduced or surfaced by the revisions)

### 1. [HIGH] W5-A contradicts W4-E's click-bound nonce model

W4-E explicitly resolves the stale-save problem by making each feedback gesture issue and immediately consume a nonce; save-time persistence is only for surface-local attributes and does not consume nonces (`02-issues.md:1686-1687`). W5-A still defines a "Save-time feedback router," diffs on save, sends feedback-shaped changes with a nonce, and treats null/missing nonce as rejection (`02-issues.md:1738`, `02-issues.md:1746`, `02-issues.md:1758-1764`, `02-issues.md:1797-1800`).

These two designs cannot both be true. Pick one. The safer fix is to keep W4-E's click-bound model and rewrite W5-A as a gesture feedback router plus a save-time surface-local-attribute router.

### 2. [HIGH] Artifact 11 retention reversal is incomplete

W0-C says all 15 artifacts landed and artifact 11 is retained (`02-issues.md:169`, `02-issues.md:175`), but the same issue still says "Artifact 11 retired" and "14 confirmed; 11 retired" (`02-issues.md:188`, `02-issues.md:193`). W5-A also says artifact-11 semantics are "retired per W0-C" while using them as the routing contract (`02-issues.md:1766`). Summary count is still impossible: "14 (01-15, including 11)" at `02-issues.md:2097`.

This is not just cosmetic; it can cause W0-C/W5-A agents to route the overlay contract away while W4-D consumes it.

### 3. [MEDIUM] W3 lane text points at the wrong inventory generation source

W1-C creates the v1.4.2 `tools/dailyos-abilities.json` with `client_side_executable` and `mcp_exposure` fields (`02-issues.md:542-546`). But W3-A and W3-C lane text still refer to the "v1.4.1 inventory artifact" (`03-wave-plan.md:349`, `03-wave-plan.md:364`). The issue bodies are better (`02-issues.md:1081`, `02-issues.md:1217`), but the wave prompts can send W3 agents to a stale artifact that lacks the Cycle-3 policy split.

## Ruling Rationale

The main architecture is converging. The cycle-2 RECONSIDER findings were largely closed correctly, with citation discipline good enough to verify. But:

- The W4 local staging block still contradicts the cross-wave dependency rule it just adopted. An agent reading only the W4 section can launch W4-A before W4-C/D.
- The W5-A save-time router contradicts the W4-E click-bound nonce model that closed the cycle-2 nonce concerns. Both cannot ship.
- Artifact 11 status is internally inconsistent within W0-C itself ("retained" and "retired" both appear), and W5-A trusts the retire branch while W4-D trusts the retain branch.
- W3 wave-prompt text still points at the v1.4.1 inventory; agents who read the wave plan first will miss the v1.4.2 policy split.

Approval at cycle-3 would ship contradictions into parallel agent execution. That is the exact failure mode L0 exists to prevent.

**Ruling:** **RECONSIDER** until the W4 staging block, W5-A feedback/nonce contract, artifact-11 stale text, and W3 inventory references are swept.

Path-α: none of the new findings are acceptance-criteria edge cases; they are execution-contract contradictions inside the plan itself, so they belong in cycle-4 closure, not maintenance.
