## STRIDE walkthrough

### EoP / gate order (§7.6)

Gate order in §7.6 matches the contract — `allowed_actors` → `required_scopes` → MCP-path `mcp_exposure` check OR client-side-JS-path `client_side_executable` check, with each rejection happening at substrate (registry or SurfaceClientBridge) before the ability body runs. Substrate-enforceable, not prose-only. The cycle-3 review's transport-prefix gates (Host→Origin→HMAC→rate-limit→auth) sit upstream of the policy gates in W2-A/W1-A0; no bypass introduced by separating `client_side_executable` from `mcp_exposure`. ADR-0111 §8 "Actor-filtered discovery extension" mirrors the same orthogonal-fields language.

### Info disclosure / introspection (§7.4 + §7.6)

§7.4 correctly splits enumeration paths:
- MCP-mediated: filter by `allowed_actors ⊇ Agent/SurfaceClient` AND `mcp_exposure ∈ {MetadataOnly, Invocable}`
- Client-side JS: filter by `allowed_actors ⊇ SurfaceClient` AND `client_side_executable = true`

`MetadataOnly` enumerates name+description+schema without registering an invocation handler — closed by construction, not by convention. Unauthorized SurfaceClient sees neither names nor schemas for abilities outside its scope grants, `mcp_exposure: None` abilities on MCP paths, or `client_side_executable: false` abilities on JS paths.

### Spoof / Tamper (ADR-0130 `ProvenanceRef`)

`Block.provenance: ProvenanceRef { invocation_id, field_path }` resolves against the canonical `AbilityOutput<Composition>.provenance` envelope; no envelope copy on the block. Preserves ADR-0102 §6 + §9 Rule 5 + ADR-0105 §8 lives-once invariant. Block-builder validation requires `field_path` to resolve into `output.provenance.field_attributions` at construction time, preventing accidental broad attribution. Resolution failure degrades visibly (provenance-unavailable banner or trust-band downgrade) — never silently to "trusted." ADR-0108 actor-filtering still applies at resolution; ref does not bypass.

### Default policy regression (§7.6 final paragraph)

Confirmed defaults: `allowed_actors = [User]`, `required_scopes = vec![]`, `mcp_exposure = McpExposure::None`, `client_side_executable = false`. Least-privilege: existing abilities not opted in are exposed on neither surface. SurfaceClient exposure is opt-in per ability with macro compile-error if `allowed_actors` includes SurfaceClient but `required_scopes` is empty AND `no_scope_required` is absent. The W1-B compile-error gate fires on actor membership regardless of `client_side_executable` value — correct, because actor membership is what makes the ability reachable at all.
