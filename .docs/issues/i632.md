# I632 — MCP query_entity reads intelligence from DB

**Area:** Backend / MCP
**Priority:** P2
**Version:** v1.0.2
**Effort:** Low (< 1 hour)
**Source:** Planning session (2026-03-21). MCP sidecar reads intelligence.json from disk — stale pattern since I513 moved intelligence to DB.

---

## Problem

The MCP sidecar's `query_entity` tool reads entity intelligence by calling `read_entity_intelligence(workspace, tracker_path)` which reads `intelligence.json` from the entity's directory on disk.

Since I513 (v1.0.0 Phase 1), the DB (`entity_assessment` table) is the source of truth for intelligence data. The `intelligence.json` file is still written as a derived output for backward compatibility, but it can be stale, missing, or out of sync with the DB.

Current behavior:
- `query_entity("Acme Corp")` → looks up account in DB ✓ → tries to read `~/Accounts/Acme Corp/intelligence.json` from disk → if file missing or stale, returns `None` for intelligence summary
- This means MCP consumers (Claude Code sessions, external tools) get incomplete intelligence when the file isn't fresh

The fix is small: read from `entity_assessment` table instead of the file.

## Solution

Update `query_entity` in `src-tauri/src/mcp/main.rs` to read intelligence from the `entity_assessment` table instead of calling `read_entity_intelligence()` which reads from disk.

## Implementation

In `src-tauri/src/mcp/main.rs`:

1. Find the `query_entity` handler (around line 217-272)
2. Replace the `read_entity_intelligence(workspace, tracker_path)` call with a DB query:
   ```rust
   // Instead of reading from disk:
   // let intel = read_entity_intelligence(&workspace, &tracker_path);

   // Read from DB:
   let assessment = db::get_entity_assessment(&conn, &entity_id)?;
   let intel_summary = assessment.map(|a| a.intelligence_data);
   ```
3. The `entity_assessment` table stores intelligence as JSON text. Parse it the same way the frontend does — the MCP response format shouldn't change, just the source.
4. Remove the `read_entity_intelligence` function if it's no longer used anywhere else. If it's used by other code paths, leave it but add a comment: `// Legacy: prefer entity_assessment table for current data`

## Acceptance Criteria

1. Call MCP `query_entity` for an enriched account. Intelligence summary returned from DB, not disk.
2. **Verification test:** Delete the `intelligence.json` file for an enriched account. Call `query_entity` again. Intelligence summary STILL returned (from DB). Before this fix, it would return `None`.
3. Call MCP `query_entity` for a person entity. Intelligence summary returned from DB.
4. Call MCP `query_entity` for an entity with no assessment data. Returns the entity info without intelligence (graceful degradation, not an error).
5. All 5 MCP tools still functional: `get_briefing`, `query_entity`, `list_entities`, `search_meetings`, `search_content`.
6. No changes to MCP response format — consumers see the same JSON structure.
