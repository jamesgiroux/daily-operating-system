# I115 — Multi-Line Action Extraction

**Status:** Open (Parking Lot)
**Priority:** P2
**Version:** Unscheduled
**Area:** Data

## Summary

The current action extraction pipeline pulls single-line action items from transcripts and inbox files. Many real-world actions span multiple lines — a task with context, a commitment with conditions, a follow-up with timeline details. The single-line constraint causes the extraction to either miss the full context or truncate it, reducing the quality of action items in the system.

This issue covers extending the extraction parser to recognize and stitch together multi-line action patterns, producing richer action records with complete context attached.

## Acceptance Criteria

Not yet specified. At minimum: multi-line action items from transcripts are extracted as a single action with the full context preserved in the action's notes or description field, not truncated at the first line break.

## Dependencies

- None blocking. Enhancement to existing `processor/` extraction pipeline.

## Notes / Rationale

Parked because single-line extraction is sufficient for the majority of action items encountered in practice. Multi-line extraction would improve quality for longer-form commitments and tasks. This was noted in early sprints as a data quality improvement but has not been prioritized against user-visible feature work.
