# I277 — Marketplace Repo for Community Preset Discoverability

**Status:** Open (Integrations Queue)
**Priority:** P3
**Version:** Unscheduled
**Area:** Integrations

## Summary

DailyOS supports custom role presets that configure the intelligence vocabulary, vitals, and prompt shapes for different user profiles. Currently, presets are shared manually. A community marketplace — a GitHub repo or similar — would let users discover, share, and import presets created by others. This would let the community build on each other's work and lower the barrier to customizing DailyOS for niche roles or industries.

The import backend already exists in DailyOS. This issue covers the community repository infrastructure and the frontend import UI (I360 covers the frontend UI specifically).

## Acceptance Criteria

Not yet specified. At minimum: a public GitHub repo with a structured preset catalog, schema validation for submitted presets, and a discovery mechanism (search/browse by role type).

## Dependencies

- I360 (Community preset import UI) is the frontend complement to this issue.
- Not version-locked; P3 priority reflects that the core product must be solid before community features make sense.

## Notes / Rationale

P3 because this is infrastructure for an ecosystem that doesn't exist yet. The preset system needs to be stable and well-understood by early users before a community repo adds value. The backend import command exists; the marketplace repo is the missing piece for discoverability.
