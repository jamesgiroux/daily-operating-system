# I141 — AI Content Tagging During Enrichment

**Status:** Open (Parking Lot)
**Priority:** P2
**Version:** Unscheduled
**Area:** Data

## Summary

During entity intelligence enrichment, the AI processes account, project, and person context to produce `intelligence.json`. This issue adds a second output to that enrichment pass: content tags — structured labels that categorize the entity's content (e.g., "renewal risk", "expansion opportunity", "technical debt", "leadership change"). These tags would enable filtering, grouping, and proactive surfacing across entities without requiring the user to manually categorize anything.

## Acceptance Criteria

Not yet specified. The core deliverable is: after enrichment, entities have a `tags` field in `entity_intel` populated with AI-generated structured labels. These labels are used somewhere in the frontend to filter or group entities.

## Dependencies

- Depends on entity intelligence pipeline (Sprint 10, I130-I138) being stable.
- Related to I359 (vocabulary-driven prompts) — role vocabulary should influence which tags are generated.

## Notes / Rationale

Parked because the primary value of entity intelligence is the narrative and signal outputs, not the tags. Tags become more valuable once there are enough entities that filtering and grouping are needed (typically 20+ accounts). Deferred until the core intelligence quality is higher and the user base warrants the filtering capability.
