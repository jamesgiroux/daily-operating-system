# I360 — Community Preset Import UI — Frontend Caller for Existing Import Backend

**Status:** Open (Parking Lot)
**Priority:** P2
**Version:** Unscheduled
**Area:** UX

## Summary

The backend already supports importing a custom role preset from a JSON file — the `import_preset` command exists and works. What's missing is a frontend UI that makes this accessible to users without requiring them to use the CLI or drag-and-drop a JSON file. This issue covers a simple import UI in Settings: browse for a preset file, validate it, import it, and activate it.

## Acceptance Criteria

Not yet specified. At minimum: an "Import Preset" button in the Settings presets section, a file picker dialog (JSON only), validation against the preset schema before import, and a confirmation screen showing what will change before the preset is activated.

## Dependencies

- The backend `import_preset` command exists (implemented in v0.11.0 as part of the preset system).
- Related to I277 (Marketplace repo) — the community repo is the source of presets users would import via this UI.

## Notes / Rationale

Parked because the preset community doesn't exist yet (I277 is also unscheduled). Building the import UI before the community repo exists is premature. Once I277 ships and there are presets to discover, this UI becomes high priority for completing the import loop.
