# I227 — Gainsight Integration — CS Platform Data Sync

**Status:** Open (Integrations Queue)
**Priority:** P2
**Version:** Unscheduled (depends on Gainsight API)
**Area:** Integrations

## Summary

Gainsight is the leading Customer Success platform used by CS teams to track health scores, NPS, renewal dates, escalations, and success plans. CS users of DailyOS maintain customer data in both Gainsight and DailyOS with no sync between them. This integration would pull Gainsight health scores, renewal dates, open risks (red accounts), and success plan milestones into DailyOS entity intelligence — so the DailyOS daily briefing and account detail pages reflect the same health picture as Gainsight, without manual duplication.

## Acceptance Criteria

Not yet specified. At minimum: connection to Gainsight API, pull of account health scores and renewal dates, mapping of Gainsight accounts to DailyOS accounts, and signal emission when health scores change.

## Dependencies

- Requires Gainsight API credentials.
- Not version-locked; pulled in when capacity and API access allow.

## Notes / Rationale

P2 because Gainsight users are a significant but not primary DailyOS user profile. The integration is valuable but less urgent than Gong (I225) for the initial user profiles being targeted. Blocked on Gainsight API access.
