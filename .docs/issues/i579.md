# I579 — Per-Email Triage Actions

**Version:** v1.0.1
**Area:** Frontend + Backend / Email
**Priority:** P1

## Problem

Users read the AI intelligence about an email but have no way to act on it. Today the only gestures are dismiss (for extracted items) and entity correction. CS users need: "I've handled this," "I need to deal with this in Gmail," and "remind me later." DailyOS is the triage layer, not the composition layer.

## Design

### Frontend

Each `EmailIntelItem` gets a hover action bar (consistent with existing dismiss X pattern):

1. **Archive** (check icon) — "I've handled this"
   - Calls existing `archive_emails` backend command (removes INBOX label in Gmail, marks `resolved_at` locally)
   - Optimistic UI: item fades out immediately
   - Undo toast for 5 seconds (calls `unarchive_email` to restore INBOX label and clear `resolved_at`)
   - After undo window, removal is permanent from the page

2. **Open in Gmail** (external link icon) — Deep link to `https://mail.google.com/mail/u/0/#inbox/{email_id}`
   - Opens in default browser via Tauri shell API
   - No state change in DailyOS

3. **Pin** (pin icon, toggleable) — "Come back to this"
   - Local-only flag, does not affect Gmail state
   - Pinned emails float to top of their score band
   - Visual indicator: subtle pin icon or left-border accent
   - Toggle: clicking again unpins

### Backend

**New migration**: Add `pinned_at DATETIME NULL` column to `emails` table.

**New commands**:
- `pin_email(email_id: String)` — sets `pinned_at = now()`
- `unpin_email(email_id: String)` — sets `pinned_at = NULL`
- `unarchive_email(email_id: String)` — adds INBOX label back via Gmail API, clears `resolved_at`

**Signal emission**:
- Pin emits signal: `email_pinned` with entity linkage (user found this important — positive relevance signal, confidence 0.7)
- Archive emits signal: `email_archived` (user processed this — neutral engagement signal)

**Sort order change**: `get_emails_enriched()` sorts pinned emails first within each score band (`pinned_at IS NOT NULL` first, then by `relevance_score` desc).

### Vocabulary (ADR-0083)

- Archive tooltip: "Archive" (matches Gmail vocabulary)
- Pin tooltip: "Pin to top"
- No system terms in UI

## Mock Data

Requires I596 (email mock data foundation). Pin scenarios:
- 2 mock emails seeded with `pinned_at` set (verify they sort to top of their band)
- Migration adds `pinned_at` column — mock data seeds must include it

## Acceptance Criteria

1. Hovering an email intel item reveals Archive, Open in Gmail, and Pin actions
2. Archive removes the email from the list (optimistic), archives in Gmail (removes INBOX label), and marks resolved in DB
3. Archive shows an undo toast for 5 seconds; undo restores INBOX label and unmarks resolved
4. Open in Gmail opens the correct thread in the user's default browser
5. Pin adds a `pinned_at` timestamp; pinned emails sort to the top of their score band
6. Pin emits a positive relevance signal to the email's linked entity
7. Unpin clears the timestamp and removes the visual indicator
8. Actions bar is keyboard-accessible (tab-focusable)
9. **Mock data**: 2 mock emails seeded as pinned; verify they appear at top of their score band in `full` scenario
