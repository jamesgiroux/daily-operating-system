# I533 ‚Äî Publication Engine: Google Drive Output Layer

**Priority:** P1
**Area:** Backend / Publication
**Version:** v1.1.0
**Depends on:** I513 (DB as sole source ‚Äî reports read from DB, not files), v1.0.0 CS report suite (I489-I491, I496-I498 ‚Äî reports must exist to publish)

## Problem

DailyOS produces intelligence that only the current user can see. Reports, health summaries, and account briefings are generated locally and displayed in the app ‚Äî there is no way to share them with a VP, colleague, or leadership team without manually exporting and emailing.

The output-layer publication model (ADR-0099 withdrawn, first-principles review affirmed) says: sharing happens at the output layer, not the data layer. DailyOS doesn't sync databases or share raw signals. It publishes **curated, finished documents** to a place the organization already governs ‚Äî Google Drive Shared folders.

### Why Google Drive, not Glean Indexing API

1. **Human-browsable.** A VP can open a Google Doc or PDF without any tool. Glean-indexed custom datasource documents are only discoverable via Glean search.
2. **Google Workspace governance applies.** DLP, retention policies, sharing controls, audit logs ‚Äî all free.
3. **Glean auto-indexes Drive.** Activity crawls every 10 minutes. Published reports become Glean-searchable within minutes ‚Äî without the Indexing API, without admin setup, without a custom connector.
4. **No admin dependency.** The Indexing API requires admin-created indexing tokens and custom datasource configuration. Google Drive just needs the user to authenticate with Google (which they already have).

### What about structured metadata?

Google Drive publication gives you human-readable documents. It does NOT give you structured, queryable metadata (e.g., "show me all accounts with health < 60"). If that's needed in v1.2.0+, a parallel push to a Glean custom datasource via the Indexing API is additive ‚Äî not a replacement for Drive publication.

## Design

### 1. Publication configuration

Users configure publication in Settings:

```rust
pub struct PublicationConfig {
    pub enabled: bool,
    pub drive_folder_id: String,       // Google Drive Shared folder ID
    pub format: PublicationFormat,      // pdf | google_doc
    pub auto_publish: bool,            // auto-publish when reports generate
    pub publish_types: Vec<String>,    // which report types to publish
}

pub enum PublicationFormat {
    Pdf,        // Upload .pdf ‚Äî static, archival
    GoogleDoc,  // Create Google Doc via Docs API ‚Äî editable, commentable
}
```

### 2. Folder structure

```
{Shared Drive}/
  DailyOS Intelligence/
    {Account Name}/
      VP Account Review ‚Äî 2026-03-04.pdf
      Renewal Readiness ‚Äî 2026-03-01.pdf
      Stakeholder Map ‚Äî 2026-02-28.pdf
    {Account Name}/
      ...
    Portfolio/
      Portfolio Health Summary ‚Äî 2026-03-04.pdf
```

- One subfolder per account (created on first publish)
- Portfolio-level reports go in a `Portfolio/` subfolder
- Filename convention: `{Report Type} ‚Äî {Date}.{ext}`
- Older versions are NOT deleted ‚Äî they accumulate for historical reference

### 3. Publication flow

```rust
pub async fn publish_report(
    report_id: &str,
    config: &PublicationConfig,
    db: &ActionDb,
    google: &GoogleApi,
) -> Result<PublicationResult, PublicationError> {
    // 1. Read report content from DB (reports table)
    // 2. Render to publication format:
    //    - PDF: generate via existing report PDF pipeline
    //    - Google Doc: convert report sections to Docs API format
    // 3. Check if account subfolder exists in Drive, create if needed
    // 4. Upload file to the correct subfolder
    // 5. Record publication event in `report_publications` table
    // 6. Return URL of published document
}
```

### 4. Storage ‚Äî `report_publications` table

```sql
CREATE TABLE report_publications (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    report_id TEXT NOT NULL,
    entity_id TEXT NOT NULL,
    entity_type TEXT NOT NULL,
    report_type TEXT NOT NULL,
    drive_file_id TEXT NOT NULL,
    drive_url TEXT NOT NULL,
    format TEXT NOT NULL,              -- 'pdf' | 'google_doc'
    published_at TEXT NOT NULL DEFAULT (datetime('now')),
    published_by TEXT,                 -- user entity ID
    FOREIGN KEY (report_id) REFERENCES reports(id)
);
```

### 5. Frontend ‚Äî Publication controls

**On report generation:** After a report generates, show "Publish" button (mono 11px, turmeric) next to the existing actions. Clicking publishes to configured Drive folder.

**In Settings:** "Publication" section:
- Google Drive folder picker (use Google Picker API or paste folder URL)
- Format toggle: PDF / Google Doc
- Auto-publish toggle per report type
- Status: "Last published: VP Account Review ‚Äî Acme Corp ‚Äî 2 hours ago"

**On report detail:** If published, show "Published" badge with Drive link. If stale (report re-generated since last publish), show "Republish" button.

### 6. Auto-publish

When `auto_publish` is enabled and a report type is in `publish_types`, the report generation pipeline calls `publish_report()` automatically after successful generation. No user action needed ‚Äî generate a VP Account Review and it appears in Drive within seconds.

### 7. Google API requirements

**Already available:** DailyOS has Google OAuth with Calendar and Gmail scopes.

**New scopes needed:**
- `https://www.googleapis.com/auth/drive.file` ‚Äî create/upload files to Drive (scoped to files created by the app, not full Drive access)
- `https://www.googleapis.com/auth/docs` ‚Äî create Google Docs (only if GoogleDoc format is used)

**APIs used:**
- Google Drive API v3: `files.create`, `files.update`, `files.list` (check folder existence)
- Google Docs API v1: `documents.create`, `documents.batchUpdate` (if GoogleDoc format)

### 8. Glean discoverability

No DailyOS code needed for Glean indexing. Once the file is in a Shared Drive that Glean's Google Drive connector indexes:
- Activity crawl picks it up within ~10 minutes
- The report content becomes searchable via Glean
- A Glean Agent (or user in Glean Chat) can query: "What's the health status of Acme?" and get the published report content
- Permissions enforced: only users with access to the Shared Drive see the results

## Acceptance Criteria

1. Configure a Google Drive Shared folder in Settings. Generate a VP Account Review. Click "Publish." Report appears as PDF in `{Shared Drive}/DailyOS Intelligence/{Account Name}/` within 10 seconds.
2. File naming follows convention: `VP Account Review ‚Äî 2026-03-04.pdf`
3. Account subfolder created automatically on first publish for that account
4. Portfolio-level reports (I491) published to `Portfolio/` subfolder
5. `report_publications` table records every publication with Drive URL
6. Auto-publish: enable for VP Account Review. Generate report. Published automatically ‚Äî no manual click.
7. Stale detection: re-generate a report. Published badge changes to "Republish" with updated content.
8. Google Drive scope request: user prompted to grant `drive.file` scope on first publication setup (incremental OAuth)
9. Glean discoverability: published report searchable in Glean within 15 minutes (depends on Glean crawl frequency ‚Äî verified manually)
10. No full Drive access ‚Äî only files created by DailyOS are accessible via `drive.file` scope

## Out of Scope

- Glean Indexing API push (structured metadata) ‚Äî v1.2.0+ enhancement if needed
- Publishing non-report content (raw intelligence, signals) ‚Äî reports only
- Unpublishing / deleting from Drive (user manages Drive directly)
- Version control in Drive (Google Drive handles version history natively)
- Notification when a report is published (Slack integration ‚Äî post-v1.1.0)
- Collaborative editing of published reports ‚Äî these are read-only outputs
