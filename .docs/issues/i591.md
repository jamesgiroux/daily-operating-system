# I591 — Unify AI Objectives with User Objectives

**Version:** v1.0.2
**Area:** Backend / Intelligence + Success Plans
**Priority:** P2

## Problem

Two parallel objective systems exist with no merge path:

1. **AI-extracted `statedObjectives`** — stored in `entity_assessment.success_plan_signals_json`, extracted from transcripts/emails during enrichment. Represents what the customer has said they want to achieve.
2. **User-created `account_objectives`** — stored in `account_objectives` table, created manually in The Work section. Represents the CS user's plan.

The AI doesn't know about the user's objectives. The user's objectives don't reference the AI's findings. The same goal can exist in both systems with no connection.

## Design

### Merge Strategy: "Adopt" Model

Don't merge the databases. Instead, let users **adopt** AI-extracted objectives into their plan.

**On enrichment**: After extracting `statedObjectives`, check each against existing `account_objectives`:
- **Match found** (fuzzy title similarity > 0.7): Update the `account_objective` with evidence from the AI extraction (add to a new `evidence_json` field). Don't modify the user's title or description.
- **No match**: Store as a suggestion candidate. Surface in The Work section as "Detected objective: [title]" with an "Adopt" button.

### Backend Changes

New column on `account_objectives`: `evidence_json TEXT` — array of evidence items from AI extraction:
```json
[
  { "source": "meeting", "source_id": "...", "date": "2026-03-10", "quote": "Customer mentioned wanting to reduce churn by 20%" }
]
```

New column on `account_objectives`: `ai_origin_id TEXT NULL` — references the original `statedObjective` ID from enrichment, enabling deduplication.

Service method `reconcile_objectives(entity_id)`:
- Called after enrichment completes for an entity
- Matches AI objectives against user objectives
- For matches: appends evidence
- For new: stores as suggestion (existing `get_objective_suggestions()` already handles this, but now with better deduplication)

### Frontend — The Work

- Existing objectives show evidence count badge: "3 mentions in calls" (clickable to see evidence)
- AI-detected objectives that don't match show as suggestion cards below user objectives
- "Adopt" button creates an `account_objective` from the suggestion, with evidence pre-populated
- "Dismiss" hides the suggestion and emits correction signal

### Intelligence Loop

- `objective_adopted` signal (confidence 0.8) — user confirmed an AI-detected objective
- `objective_suggestion_dismissed` signal (confidence 0.3) — feeds Bayesian learning
- Evidence accumulation makes objectives more credible over time — an objective mentioned in 5 meetings is stronger than one mentioned once

## Acceptance Criteria

1. AI-extracted objectives are matched against existing user objectives (fuzzy match > 0.7)
2. Matched objectives get evidence appended (source, date, quote) without modifying user's title
3. Unmatched AI objectives surface as suggestions in The Work section
4. "Adopt" creates an account_objective with pre-populated evidence
5. "Dismiss" emits correction signal for Bayesian learning
6. Objectives show evidence count ("mentioned in 3 calls")
7. Evidence detail is viewable (list of sources with dates and quotes)
8. Duplicate suggestions suppressed (same objective not suggested twice)
