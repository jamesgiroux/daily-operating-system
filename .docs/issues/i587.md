# I587 — Action Context in Meeting Prep

**Version:** v1.0.2
**Area:** Backend / Intelligence + Frontend / Meeting Detail
**Priority:** P1

## Problem

Meeting prep includes `open_actions` for the account but only passes title and priority. A CS user walking into a meeting should know: "2 actions overdue for Acme, 1 waiting on Sarah for 5 days, objective 'Close Renewal' is 60% complete." Today they get a flat list of action titles. The meeting prep system has the data — it just doesn't assemble the full picture.

## Design

### Backend — Richer Action Context

Extend `gather_account_context()` in `prepare/meeting_context.rs`:

Replace flat `open_actions` with structured `ActionContext`:
```rust
struct ActionContext {
    pending_count: usize,
    overdue_count: usize,
    waiting_count: usize,
    overdue_actions: Vec<ActionSummary>,      // title, days_overdue, priority
    waiting_actions: Vec<WaitingSummary>,      // title, waiting_on, days_waiting
    recent_completions: Vec<CompletionSummary>, // title, completed_at (last 14 days)
    objective_progress: Vec<ObjectiveProgress>, // objective_title, completion_pct, milestone_summary
}
```

Queries needed:
- `get_overdue_actions_for_entity()` — pending actions past due_date
- `get_waiting_actions_for_entity()` — actions with `waiting_on` set, with age
- `get_recent_completions_for_entity(14)` — completed in last 14 days
- `get_objective_progress_for_entity()` — active objectives with milestone completion percentage

### Backend — Enrichment Prompt Update

In `intelligence/prompts.rs`, update the meeting prep prompt section for actions:
- Include overdue actions by name ("OVERDUE: Send proposal — 3 days late, P1")
- Include waiting status ("WAITING: Sarah to review contract — 5 days")
- Include objective progress ("Objective 'Close Renewal': 2/4 milestones done")
- Include recent wins ("COMPLETED last 2 weeks: Delivered POC results, Scheduled exec review")

This gives the LLM enough context to generate prep notes like: "You have 2 overdue commitments to Acme — address these before discussing expansion."

### Frontend — Meeting Detail

In the meeting detail prep section, render action context as a structured block:
- **Overdue** (red accent): list of overdue actions with days overdue
- **Waiting on** (amber accent): delegated actions with waiting duration
- **Recent wins** (green accent): completions in last 14 days
- **Objective progress**: progress bars or fraction display per active objective
- Section hidden when no actions exist for the account

### Frontend — Briefing

On the daily briefing, meetings with overdue actions for their account should get a subtle flag: "2 overdue actions" in the meeting card meta line.

## Acceptance Criteria

1. Meeting prep context includes overdue count, waiting count, and recent completions — not just flat action titles
2. Overdue actions listed by name with days overdue and priority in prep context
3. Waiting-on actions listed with delegation target and waiting duration
4. Recent completions (last 14 days) included as positive context
5. Active objective progress (milestone completion %) included in prep context
6. Meeting detail page renders structured action context (overdue/waiting/wins/progress)
7. Daily briefing meeting cards show overdue action count for the account
8. LLM meeting prep notes reference action state when relevant (verified with real data)
