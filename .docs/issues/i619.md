# I619 — Prompt evaluation suite

**Area:** Backend / Testing / Intelligence
**Priority:** P1
**Version:** v1.0.1
**Source:** v1.0.0 GA engineering review (2026-03-16), Issue 10A

---

## Problem

Phase 4b shipped significant prompt changes (6 win sub-types, Red/Yellow/Green risk urgency, CHAMPION_HEALTH extraction, COMMITMENTS, interaction dynamics). Parsing tests exist (117 in intelligence/, 33 in transcript) but they test JSON shape, not content quality. There is no eval suite to validate prompt output quality against baselines.

Prompt regressions are silent: code compiles, JSON parses, but "Customer seems happy" starts being extracted as a win again, or champion health stops being flagged as RED when a champion departs.

## Scope

### Golden Fixtures

Create golden input/output pairs in `src-tauri/src/intelligence/fixtures/` (new directory):

1. **Transcript fixtures** (input: transcript text, expected: structured extraction)
   - `transcript_cs_renewal.txt` — CS renewal call with champion health, commitments, risks
   - `transcript_cs_onboarding.txt` — Onboarding call with adoption signals, action items
   - `transcript_executive_review.txt` — QBR with expansion signals, competitive mentions
   - Expected outputs as `.json` files with minimum required fields

2. **Intelligence response fixtures** (input: mock LLM response, expected: parsed assessment)
   - `enrichment_response_full.json` — Complete 6-dimension response
   - `enrichment_response_sparse.json` — Minimal data (1 meeting, no email)
   - `enrichment_response_malformed.json` — Partial/broken JSON for error handling

3. **Dimension prompt fixtures** (input: context, expected: prompt contains key sections)
   - One per dimension: competitive, stakeholder, support, engagement, financial, strategic

#### Category 2: Response Parsing Tests (~8 tests)

Verify golden fixtures parse correctly and produce expected assessment structures.

```rust
#[test]
fn test_parse_full_enrichment_response() {
    let response = include_str!("fixtures/enrichment_response_full.json");
    let (assessment, relationships) = parse_intelligence_response(response).unwrap();

    assert!(assessment.risks.is_some());
    assert!(assessment.wins.is_some());
    assert!(assessment.dimensions_blob().is_some());

#[test]
```

#### Category 3: Transcript Extraction Quality Tests (~6 tests)

Verify golden transcripts produce correctly categorized outputs.

```rust
#[test]
fn test_transcript_extraction_wins_have_subtypes() {
    let response = include_str!("fixtures/transcript_cs_renewal_response.json");
    let parsed = parse_transcript_response(response).unwrap();

    // I554: wins must have sub-types, not generic
    for win in &parsed.wins {
        assert!(
            ["ADOPTION", "EXPANSION", "RETENTION", "ADVOCACY", "MILESTONE", "VALUE_REALIZATION"]
                .contains(&win.sub_type.as_str()),
            "Win missing sub-type: {:?}", win
        );
    }
}

#[test]
fn test_transcript_extraction_risks_have_urgency() {
    let response = include_str!("fixtures/transcript_cs_renewal_response.json");
    let parsed = parse_transcript_response(response).unwrap();

    for risk in &parsed.risks {
        assert!(
            ["RED", "YELLOW", "GREEN_WATCH"].contains(&risk.urgency.as_str()),
            "Risk missing urgency: {:?}", risk
        );
    }
}

#[test]
fn test_champion_departure_flagged_as_red() {
    let response = include_str!("fixtures/transcript_champion_departure_response.json");
    let parsed = parse_transcript_response(response).unwrap();

    let champion_health = parsed.champion_health.as_ref().unwrap();
    assert_eq!(champion_health.status, "lost");
    // Should have corresponding RED risk
    assert!(parsed.risks.iter().any(|r| r.urgency == "RED"));
}

#[test]
fn test_generic_sentiment_not_extracted_as_win() {
    // "Customer seems happy" should NOT be a win
    let response = include_str!("fixtures/transcript_generic_sentiment_response.json");
    let parsed = parse_transcript_response(response).unwrap();

    for win in &parsed.wins {
        assert!(
            !win.description.to_lowercase().contains("seems happy"),
            "Generic sentiment extracted as win: {:?}", win
        );
    }
}
```

#### Category 4: Dimension Merge Tests (~4 tests)

Verify dimension results merge correctly into existing assessments.

```rust
#[test]
fn test_merge_dimension_preserves_existing() {
    let mut existing = sample_assessment();
    let new_competitive = sample_dimension_result("competitive");

    merge_dimension_into(&mut existing, &new_competitive);

    // Competitive updated
    assert!(existing.dimensions_blob().unwrap().contains("competitive"));
    // Other dimensions preserved
    assert!(existing.risks.is_some()); // Not overwritten
}
```
