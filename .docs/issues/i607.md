# I607 — Weekly Impact v2

**Version:** v1.0.3 Phase 4
**Depends on:** I604 (schema, user_weekly_summary table, animation primitives)
**Type:** Enhancement
**Scope:** Backend (reports/weekly_impact, scheduler) + Frontend (WeeklyImpactPage, slide components)

## Problem

Weekly Impact runs a single monolithic PTY call at report-open time, dumping raw `signal_events`, `meeting_transcripts`, `captures`, and `actions` into one prompt (~3-8K tokens). It ignores `entity_assessment` entirely and produces no comparative framing. The `intel_hash` is a date string that can never be invalidated by new data.

With `user_weekly_summary`, the report becomes display-only. Mechanical fields are computed at week rollover; the two editorial fields (`headline`, `what_you_did`) are pre-generated by a background PTY call Sunday night or Monday morning.

## Design

### Weekly Rollover Task

A new scheduler task that fires Sunday evening (or on first Monday app open):

**Mechanical computation (instant, no PTY):**
- `total_meetings` — `COUNT(*)` from `meetings` for the prior week
- `total_actions_closed` — `COUNT(*)` from `actions.completed_at` in range
- `priorities_moved_json` — cross-join user priorities against `meeting_entities` + `captures` for priority-linked entities. Each item: `{priorityText, whatHappened, source}`
- `wins_json` — `captures` where `capture_type LIKE '%win%'` or positive urgency, week range
- `watch_json` — `captures` where `urgency = 'red'` or `'yellow'`, week range
- `into_next_week_json` — open actions due next week + watch items carried forward
- `meetings_by_day_json` — `[Mon, Tue, Wed, Thu, Fri]` meeting counts for sparkline
- `prior_week_headline` — read from prior week's `user_weekly_summary` row

**AI computation (one background PTY call, `ModelTier::Extraction`, 30s timeout):**
- `headline` — one-liner with comparative framing (receives prior week headline + this week's stats)
- `what_you_did` — 2-sentence prose summary of the week's activity

The PTY prompt is tiny (~500-800 tokens): just the mechanical stats + prior week context + "write a headline and summary." Fast, focused, cheap.

**Storage:** Upsert to `user_weekly_summary` keyed on `week_start`.

**Entity intelligence integration:** The `priorities_moved` computation queries `entity_assessment.executive_assessment` for accounts linked to priorities — this gives the AI-synthesized "what happened" narrative rather than raw signal dumps.

### Report Display

The report command reads from `user_weekly_summary`. If the row exists, renders instantly. If not (first week, app just installed), shows a "Your weekly impact will be ready Monday morning" placeholder — no on-demand PTY fallback.

Schema additions to `WeeklyImpactContent`:
- `weekOverWeekMeetings: number | null` — delta vs prior week (computed from two summary rows)
- `weekOverWeekActions: number | null`
- `priorWeekHeadline: string | null`
- `meetingsByDay: number[]` — [Mon..Fri] for sparkline
- `priorityAlignmentPct: number | null`

### Frontend Delight

- **Cover slide:** `AnimatedCounter` for totalMeetings and totalActionsClosed. Week-over-week delta with directional arrow.
- **Priorities slide:** `ProgressBar` per priority showing alignment.
- **The Work slide:** `Sparkline` for Mon-Fri meeting distribution.
- **Watch slide:** Staggered reveal for risk items.
- **All slides:** `useSlideReveal` triggers content entry.
- **No "Generating..." state** — content is pre-computed, renders instantly.
- **Animation timing:** Fast (0.3s) — crisp, operational mood. `cubic-bezier(0.22, 1, 0.36, 1)`.

### Comparative Framing

Week-over-week comparisons are computed mechanically:
- `weekOverWeekMeetings = this_week.total_meetings - prior_week.total_meetings`
- The `headline` field already receives prior week context during PTY generation
- Frontend displays deltas with directional indicators

## Acceptance Criteria

1. Zero PTY calls when opening the Weekly Impact report — content from `user_weekly_summary`
2. Rollover task computes mechanical fields and generates headline/what_you_did via one focused PTY call (under 1000 tokens)
3. Report renders instantly — no "Generating..." state
4. `AnimatedCounter` on cover stats counts up on slide entry
5. `Sparkline` shows 5-day meeting distribution
6. Week-over-week delta displayed when prior week data exists
7. Priorities moved references `entity_assessment.executive_assessment` for context (not raw signals)
8. `prefers-reduced-motion` disables all animations
9. Graceful empty state when no summary exists for the requested week
10. In-place editing preserved — edits stored in `user_weekly_summary`
