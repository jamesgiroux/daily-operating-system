# I613 — hygiene.rs decomposition into sub-modules

**Area:** Code Quality / Backend
**Priority:** P2
**Version:** v1.0.1
**Source:** v1.0.0 GA engineering review (2026-03-16)

---

## Problem

`hygiene.rs` is 3,463 lines — the largest file in the Rust backend. While functionally cohesive (single domain: data quality), the monolithic structure has two concrete downsides:

1. **No error isolation**: The 160-line `run_hygiene_scan()` orchestrator runs 15+ fix operations sequentially. If phase 1b fails (e.g., `archive_phantom_accounts` hits a constraint violation), phase 2 matching still runs on potentially inconsistent state.

2. **Reusable logic buried**: `score_name_similarity()` (70 lines of Jaro-Winkler + prefix bonus + case normalization) is useful beyond hygiene but inaccessible without importing the entire module.

No bugs reported. The module works correctly. This is maintainability and testability improvement.

## Decomposition Plan

```
src-tauri/src/hygiene.rs (3,463 lines)
    ↓
src-tauri/src/hygiene/
├── mod.rs              — re-exports + run_hygiene_scan() orchestrator + run_hygiene_loop()
├── detectors.rs        — Gap detection functions
│   ├── check_upcoming_meeting_readiness()
│   ├── detect_duplicate_people()
│   └── build_intelligence_hygiene_status()
├── fixers.rs           — Phase 1/1b mechanical fixes
│   ├── fix_unknown_relationships()
│   ├── backfill_file_summaries()
│   ├── fix_meeting_counts()
│   ├── fix_renewal_rollovers()
│   ├── retry_abandoned_quill_syncs()
│   ├── archive_phantom_accounts()
│   ├── relink_orphan_internal_accounts()
│   └── archive_empty_shell_accounts()
├── matcher.rs          — Phase 2 entity matching + enrichment orchestration
│   ├── fix_auto_merge_duplicates()
│   ├── fix_co_attendance_links()
│   ├── detect_low_confidence_matches()
│   ├── resolve_names_from_calendar()
│   ├── resolve_names_from_emails()
│   ├── auto_link_people_by_domain()
│   ├── enqueue_ai_enrichments()

│   └── score_name_similarity()  (candidate for extraction to util.rs)
└── narrative.rs        — Report building + UI construction
    ├── build_hygiene_narrative()
    ├── run_overnight_scan()
    ├── join_prose_list()
    └── hygiene_gap_action()
```

## Scope

1. Create `src-tauri/src/hygiene/` directory with 5 files
2. Move functions to their sub-module per the plan above
3. `mod.rs` re-exports all public functions (no import changes needed for callers)
4. Add error isolation in `run_hygiene_scan()`: wrap each phase in a `match` that logs errors and continues to next phase
5. Move tests alongside their functions

## Acceptance Criteria

1. `wc -l src-tauri/src/hygiene/mod.rs` < 300 (orchestrator + re-exports only)
2. Each sub-module is independently testable
3. `run_hygiene_scan()` catches phase-level errors: a failure in phase 1b does not skip phase 2
4. All existing tests pass with zero changes to test assertions
5. `cargo clippy -- -D warnings` clean
6. No caller changes needed — `use crate::hygiene::*` still works

## Out of Scope

- Adding new hygiene fix types
- Signal emission from hygiene operations
- Changing hygiene scheduling or scan interval logic
