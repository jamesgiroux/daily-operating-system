# I577 — Reply Debt Surface

**Version:** v1.0.1
**Area:** Backend + Frontend / Email
**Priority:** P1

## Problem

The most damaging CS behavior is silence — a customer emails you and gets no response. DailyOS already tracks `user_is_last_sender` and thread timestamps in the `emails` table, but never surfaces "you owe this customer a reply." Reply debt is invisible until the customer escalates.

## Design

### Backend

**Query** (`db/emails.rs`): New function `get_reply_debt_emails()` — active emails where `user_is_last_sender = false` AND `entity_id IS NOT NULL`, ordered by age (oldest unanswered first). Exclude noise senders (automated, noreply, etc.) using existing classification.

**New struct** `ReplyDebtItem`:
- `email_id`, `thread_id`
- `entity_id`, `entity_name`, `entity_type`
- `sender`, `sender_email`
- `subject`
- `received_at` (timestamp of the email)
- `hours_since_received` (computed)
- `entity_health_declining` (bool — flag if entity engagement dimension trend is "declining")

**New field on `EmailBriefingData`**: `reply_debt: Vec<ReplyDebtItem>`

**Signal emission**: New signal type `reply_debt_aging` emitted when an unanswered customer email crosses 48h and 72h thresholds. Emitted by the email sync/enrichment pipeline (not on every page load). Signal includes `hours_overdue` in metadata. Only emits once per threshold per email (track via `email_signals` deduplication).

**Intelligence Loop integration**: `reply_debt_aging` signal feeds the engagement health dimension. When reply debt is cleared (email archived or user detected as last sender on next sync), emit `reply_debt_cleared` positive signal.

### Frontend

New section on The Correspondent between HERO and YOUR MOVE — **"AWAITING YOUR REPLY"**.

- Items sorted oldest-first (most urgent debt at top)
- Each item: entity name, sender name, subject (truncated), human-readable age ("2 days ago")
- Entity name links to account detail
- "Open in Gmail" action (deep link to thread)
- "Handled" action — marks resolved locally, does NOT archive in Gmail (user may have already replied outside DailyOS)
- If entity engagement is declining, show subtle indicator (e.g., "engagement declining" in meta line)
- Section hidden when empty

### Vocabulary (ADR-0083)

- Section heading: "AWAITING YOUR REPLY" (not "reply debt")
- No system terms exposed — "engagement declining" not "engagement dimension score decreasing"

## Mock Data

Requires I596 (email mock data foundation). Specific reply debt scenarios:
- Acme: 2 emails where `user_is_last_sender = false` — one 1 day old (normal), one 3 days old (aging, should trigger 48h signal)
- Globex: 1 email 5 days old with entity health declining (compound signal)
- Initech: 0 reply debt (clean state — verifies section hidden)

## Acceptance Criteria

1. Emails where customer was last sender appear in "AWAITING YOUR REPLY" section, sorted oldest-first
2. Each item shows entity name, sender, subject, and human-readable age ("2 days ago")
3. Items only appear for entity-linked emails (no noise senders)
4. Clicking "Open in Gmail" opens the correct thread in the user's default browser
5. "Handled" action removes from the section and emits a positive engagement signal
6. After 48h unanswered, a `reply_debt_aging` signal is emitted to the entity
7. Signal feeds the engagement health dimension — verified via entity health recalculation
8. Section does not appear when there are no items
9. **Mock data**: `full` scenario seeds reply debt emails for Acme (1d + 3d) and Globex (5d); Initech has none. Section renders correctly with mock data.
