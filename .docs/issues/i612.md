# I612 — InboxPage + AccountsPage inline style migration to CSS modules

**Area:** Frontend / Code Quality
**Priority:** P2
**Version:** v1.0.1
**Source:** v1.0.0 GA engineering review (2026-03-16)
**Related:** I447 (design token audit), I545 (entity detail style migration), I542 (MeetingDetail style migration)

---

## Problem

InboxPage (58 `style={{}}` instances) and AccountsPage (51 `style={{}}` instances) are the two highest-traffic pages with the most inline style debt. Phase 3 targeted MeetingDetailPage and settings components (now clean). These two pages were not in scope but represent the next largest pockets of inline styling.

Inline styles bypass the design token system (ADR-0073), making future theme support impossible for these pages and increasing per-render object allocation in React.

## Scope

1. **InboxPage.tsx** — extract 58 inline styles to `InboxPage.module.css`
   - Replace hardcoded colors with design token custom properties
   - Replace hardcoded spacing with token values
   - Preserve all visual behavior (pixel-perfect match)

2. **AccountsPage.tsx** — extract 51 inline styles to `AccountsPage.module.css`
   - Same approach: tokens for colors, spacing, typography
   - Account card grid, health badge, filter controls

3. **Verify**: `grep -n 'style={{' src/pages/InboxPage.tsx src/pages/AccountsPage.tsx` returns zero results

## Acceptance Criteria

1. Zero `style={{}}` in InboxPage.tsx and AccountsPage.tsx
2. All extracted styles use design token custom properties (no hardcoded hex, rgba, or px values where tokens exist)
3. Visual diff: screenshot before vs. after shows no regressions (use `/browse` for verification)
4. No new CSS files beyond the two page modules — reuse existing shared styles where applicable
5. Both pages render correctly with mock data and production-shape data

## Out of Scope

- Refactoring page component structure (layout changes)
- Migrating other pages' inline styles (each page is its own issue if needed)
- Creating new design tokens — use existing tokens from `design-tokens.css`
