# I628 — Success plan milestone auto-completion from lifecycle events

**Area:** Backend / Success Plans + Signals
**Priority:** P2
**Version:** v1.1.0
**Effort:** Small-Medium
**Depends on:** I623 (lifecycle intelligence engine)
**Source:** CEO plan review (2026-03-21)

---

## Problem

Success plans have milestones tied to lifecycle stages (e.g., Renewal Preparation plan has "Contract signed" milestone). When the lifecycle engine (I623) detects a renewal and transitions the account to "active," the corresponding success plan milestone should auto-complete. Currently, lifecycle events and success plan milestones are disconnected — a CSM would have to manually check off the milestone after the system already detected the event.

This is exactly the double-work pattern DailyOS is designed to eliminate. The system knows the event happened. The milestone matches the event. The AI should close the loop.

## Design Principle

**Silent auto-completion.** The system completes matching milestones without asking. No confirmation modal. No "did you mean?" prompt. The timeline shows what was completed and why. If the auto-completion was wrong, the CSM unchecks it — same correction pattern as any other AI action.

## Solution

### Matching Engine

When `apply_lifecycle_transition()` fires (I623), after the transition is applied:

1. Query active success plans for the account
2. For each plan, check milestones against the transition type
3. Auto-complete milestones with confidence ≥ 0.8
4. Log matches at 0.5-0.8 confidence in the timeline (but don't auto-complete)

### Matching Rules

| Lifecycle Event/Transition | Milestone Keywords | Template | Confidence |
|---|---|---|---|
| renewing → active (renewal confirmed) | "contract signed," "renewal," "agreement" | Renewal Preparation | 0.95 |
| onboarding → active (go-live) | "go-live," "launch," "production" | Onboarding-Standard | 0.9 |
| onboarding → active (via kickoff + time) | "onboarding complete," "setup complete" | Onboarding-Standard | 0.85 |
| any → at_risk (escalation) | "escalation acknowledged" | At-Risk Recovery | 0.8 |
| at_risk → active (recovery) | "recovery," "confidence restored" | At-Risk Recovery | 0.8 |
| ebr_completed event | "EBR," "executive review" | Growth & Expansion | 0.9 |
| qbr_completed event | "QBR," "quarterly review" | Growth & Expansion | 0.9 |

Matching is keyword-based against milestone `name` field (case-insensitive, partial match). The confidence reflects how unambiguous the match is.

### Auto-Completion Record

When a milestone is auto-completed:

```rust
// In services/success_plans.rs
fn auto_complete_milestone(
    plan_id: &str,
    objective_id: &str,
    milestone_id: &str,
    trigger_type: &str,      // "lifecycle_transition" or "lifecycle_event"
    trigger_detail: &str,     // e.g., "renewing → active via email signal"
    confidence: f64,
) -> Result<()>;
```

The milestone record is updated:
- `completed = true`
- `completed_at = now()`
- `completed_by = "system"` (vs "user" for manual completions)
- `completion_trigger = trigger_detail`

### Timeline Entry

The account timeline shows:
```
Mar 15 · Milestone completed
"Contract signed" (Renewal Preparation plan)
— triggered by lifecycle transition: renewing → active
┊ via email signal · auto-completed
```

### Cascade Check

After milestone completion, check if all milestones in the objective are complete → auto-complete the objective. If all objectives complete → mark the plan as completed. The full cascade: event → milestone → objective → plan.

## Acceptance Criteria

1. Account transitions from "renewing" to "active" (I623 lifecycle engine). Active Renewal Preparation success plan's "Contract signed" milestone auto-completes. No user action required.
2. Auto-completed milestone shows `completed_by = "system"` and `completion_trigger` referencing the lifecycle transition.
3. Account timeline entry shows: milestone name, plan name, trigger description, provenance.
4. Milestone at 0.5-0.8 confidence match is logged in timeline but NOT auto-completed: "Possible match: {milestone} in {plan} — {confidence}. [Complete] [Dismiss]"
5. Objective auto-completes when all its milestones are complete (including auto-completed ones).
6. Plan auto-completes when all objectives are complete.
7. Success plan completion percentage updates after auto-completion.
8. CSM can uncheck an auto-completed milestone — unchecking emits `user_correction` signal.
9. Multiple lifecycle transitions in sequence correctly cascade: onboarding → active completes Onboarding milestones; later renewing → active completes Renewal milestones. No cross-plan contamination.
