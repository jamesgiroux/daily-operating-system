# I597 — Action & Success Plan Mock Data Expansion

**Version:** v1.0.2
**Area:** Backend / Dev Tools
**Priority:** P0 (prerequisite for all v1.0.2 action features)

## Problem

Current action mock data only has 10 rows, all in "pending" status. No proposed, completed, waiting, or archived actions exist. Success plan data is moderate but missing commitment linkage, value_delivered persistence, and action-milestone links. Every v1.0.2 feature needs richer mock data to develop and verify against.

## Acceptance Criteria

1. `full` scenario seeds 30+ actions across proposed/pending/completed/waiting/stale states
2. Completed actions distributed over 30 days for velocity calculation (at least 8)
3. 3 waiting-on actions with varied delegation age and overdue status
4. 3 aging pending actions (14d, 25d, 35d) for zero-guilt cleanup scenarios
5. 2 decision-requiring actions with `needs_decision = 1`, owner, and stakes
6. 8+ action-objective-milestone links (including completed→milestone cascade scenarios)
7. `value_delivered_json` seeded on Acme (2 items, 1 confirmed) and Globex (1 item)
8. `recommended_actions` seeded on Acme (2) and Globex (2)
9. 8 captured commitments (3 existing + 5 new), 2 with milestone linkage
10. `rejected_action_patterns` seeded with 2 suppression patterns
11. 13+ action-related signal_events rows
12. `purge_mock_data()` cleans all new mock data
13. Actions page renders all tabs (Suggested/Pending/Completed + Delegated) from mock data without errors
14. Momentum section (I589) shows meaningful velocity from mock completed actions
