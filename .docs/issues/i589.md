# I589 — Completion Momentum Surface

**Version:** v1.0.2
**Area:** Frontend + Backend / Actions
**Priority:** P1

## Problem

Actions are completed and vanish after 48 hours. There's no sense of progress, no visible momentum. CS users need to feel that their work is accumulating into something — not disappearing into a void. The zero-guilt design means: you should feel good about what you've done, not anxious about what's left.

## Design

### Backend — Completion Analytics

New query `get_action_momentum(days: i64)` in `db/actions.rs`:
```rust
struct ActionMomentum {
    completed_this_week: usize,
    completed_last_week: usize,
    trend: String,                    // "accelerating" | "steady" | "slowing"
    accounts_progressed: Vec<String>, // account names with completions
    longest_streak: usize,            // consecutive days with at least 1 completion
    current_streak: usize,
    avg_days_to_complete: f64,
    overdue_cleared: usize,           // overdue actions completed (extra credit)
}
```

New field on actions page data: `momentum: ActionMomentum`.

### Frontend — Momentum Section

**Actions Page**: New section at top of page (before the action list), editorial style:
- Headline: dynamic based on trend
  - Accelerating: "Picking up steam — {N} actions closed this week"
  - Steady: "{N} actions closed this week across {M} accounts"
  - Slowing: "Quiet week — {N} actions closed" (no judgment, zero-guilt)
- Stat line: "{completed_this_week} this week · {accounts_progressed.length} accounts · {current_streak}-day streak"
- Section collapses to a single stat line if user has seen it (localStorage flag)

**Daily Briefing**: If yesterday had completions, brief mention: "Yesterday: completed {N} actions for {accounts}." Gives the user a micro-win before diving into today.

### Personality Integration

Use existing personality system for copy variations:
- Professional: "12 actions completed this week across 4 accounts"
- Warm: "Strong week — 12 actions knocked out, 4 accounts moving forward"
- Direct: "12 done. 4 accounts progressed."

### Intelligence Loop

No new signals needed — this surfaces data already captured by `action_completed` signals. It's a read-only aggregation surface.

## Mock Data

Requires I597 (action mock data expansion). Uses 8 completed actions distributed over 30 days:
- Acme: 5 completions (strong velocity → "accelerating" or "steady")
- Globex: 2 completions (sparse)
- Initech: 1 completion
- Should show: "8 actions closed this week across 3 accounts" (or similar based on date distribution)

## Acceptance Criteria

1. Actions page shows momentum section with completion count, trend, and account names
2. Trend accurately reflects week-over-week velocity (accelerating/steady/slowing)
3. Current streak and longest streak calculated correctly
4. Daily briefing mentions yesterday's completions when > 0
5. Momentum copy uses personality system for tone
6. Section is non-judgmental for low-activity periods (zero-guilt principle)
7. Momentum section collapses after first view (doesn't dominate the page)
8. **Mock data**: Momentum section renders meaningful stats from 8 completed mock actions. Trend calculation produces a non-zero result.
