# I618 — Smoke tests for top 5 mutation services

**Area:** Backend / Testing
**Priority:** P1
**Version:** v1.0.1
**Source:** v1.0.0 GA engineering review (2026-03-16), Issue 8A

---

## Problem

16/20 service layer modules have zero unit tests. The service layer is the mandatory mutation boundary (ADR I512) — every user-facing write goes through here. A wrong signal emission in `mutations.rs` would silently corrupt health scoring with no test to catch it.

## Scope — 5 Services, ~25 Smoke Tests

### Test Infrastructure

Tests go in each service file's `#[cfg(test)] mod tests { }` block. Key patterns:

```rust
#[cfg(test)]
mod tests {
    use super::*;
    use crate::db::core::test_utils::test_db;

    #[test]
    fn test_upsert_account_emits_signal() {
        let db = test_db();
        // Arrange: seed minimal required data
        // Act: call service function
        // Assert: verify DB state + signal_events row
    }
}
```

**Signal verification pattern**:
```rust
let signals: Vec<String> = db.conn.prepare(
    "SELECT signal_type FROM signal_events WHERE entity_id = ?1"
)?.query_map(params![entity_id], |row| row.get(0))?.collect();
assert!(signals.contains(&"entity_updated".to_string()));
```

## Acceptance Criteria

1. ~25 smoke tests across 5 service modules, all passing
2. Every mutation test verifies both DB state AND signal emission
3. `test_dismiss_intelligence_item` confirms `intelligence_curated` signal, NOT `user_correction` (I530 taxonomy)
4. `test_update_intelligence_field` confirms `user_correction` signal with source penalization
5. Tests use `test_db()` — no external dependencies, no PTY, no network
6. `cargo test` all pass, `cargo clippy -- -D warnings` clean
