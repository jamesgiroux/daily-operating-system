## Problem

Two schema integrity gaps in the migration system:

1. **Missing integrity checks**: `verify_required_schema()` in `migrations.rs` checks columns added in migrations 055-059 and 063, but skips migrations 060 (`dimensions_json`) and 068 (`success_plan_signals_json`). If either migration was interrupted, the schema is silently broken â€” `upsert_entity_intelligence()` crashes at runtime.

2. **Migration 068 bug**: Line 3 uses `CREATE TABLE IF NOT EXISTS entity_assessment` which silently no-ops since the table already exists (created in 055, rebuilt in 058). The intended table rebuild with `success_plan_signals_json` never executes for existing databases. Compare with migration 058 which correctly uses the DROP+CREATE rebuild pattern.

## Scope

### Part A: Add integrity checks (migrations.rs)

Add version-gated checks in `verify_required_schema()` after the existing `version >= 63` block:

```rust
if version >= 60 {
    let assessment_cols = table_columns(conn, "entity_assessment")?;
    if !assessment_cols.contains("dimensions_json") {
        return Err(
            "Schema integrity check failed: missing column entity_assessment.dimensions_json"
                .to_string(),
        );
    }
}

if version >= 68 {
    let assessment_cols = table_columns(conn, "entity_assessment")?;
    if !assessment_cols.contains("success_plan_signals_json") {
        return Err(
            "Schema integrity check failed: missing column entity_assessment.success_plan_signals_json"
                .to_string(),
        );
    }
}
```

### Part B: Fix migration 068

Rewrite `068_success_plans.sql` to follow the 058 pattern:
1. Remove the buggy `CREATE TABLE IF NOT EXISTS entity_assessment` on line 3
2. Wrap in `BEGIN IMMEDIATE` / `COMMIT`
3. Use `DROP TABLE IF EXISTS entity_assessment_new` + `CREATE TABLE entity_assessment_new` (with all columns including `success_plan_signals_json`)
4. `INSERT OR REPLACE INTO entity_assessment_new SELECT ... FROM entity_assessment` â€” include `success_plan_signals_json` in the column list (populated as NULL for existing rows)
5. `DROP TABLE entity_assessment` + `ALTER TABLE entity_assessment_new RENAME TO entity_assessment`
6. Recreate indexes

**Reference**: `058_health_schema_evolution.sql` lines 1-112 for the correct pattern.
