# I646 - Source-attributed account detail + meeting briefing refresh

**Area:** Frontend / Account Detail + Meeting Detail + Backend detail assembly
**Priority:** P1
**Version:** v1.1.0 Phase 5
**Effort:** Large
**Depends on:** I644, I645, existing dossier architecture (ADR-0084), I625 provenance markers
**Source:** Globex Holdings Glean research. A user found a simple Glean snapshot more useful than the current account page because it surfaced richer commercial reality, stakeholder coverage, operational friction, and source-rich context.

---

## Problem

DailyOS account detail is already thoughtful, but it is still leaving high-value account understanding on the table.

Today:

- the hero / vitals strip is effectively user-entry-first and does not automatically benefit from richer Glean-backed facts
- core commercial, relationship, and support truths are too diffuse
- source attribution is present but not yet strong enough for verification
- older issues can read as equally relevant to current issues if timeliness is not enforced

The result is a page that is elegant but not yet the best possible operating surface for an RM / CSM / TAM.

---

## Solution

Refresh account detail and meeting detail around the new source-aware DB truth.

### 1. Vitals strip becomes fact-reinforced

Allow the hero to auto-populate or refresh high-confidence factual vitals when the user has not explicitly overridden them.

Good candidates:

- ARR
- lifecycle
- renewal date
- renewal likelihood
- support package / tier
- active subscriptions
- growth potential / ICP fit when appropriate

Rules:

- if the field is empty, promote the best factual value
- if the field is source-backed and unchanged by the user, refresh it from stronger or newer evidence
- if the user edited it, keep the user value and show the conflicting source suggestion instead of overwriting

### 2. Account detail chapters become more source-aware

Use the existing dossier chapters more intentionally.

Recommended chapter emphasis:

- `The Headline`
  - what matters right now
  - latest source-backed headline with provenance
- `Outlook`
  - renewal and commercial reality
  - ARR, renewal likelihood, opportunity stage, recommended start date, pricing method, procurement complexity
- `State of Play`
  - current themes, technical footprint, product/add-on context, adoption and support summary
- `The Room`
  - champions, decision makers, procurement, security, internal team, coverage assessment
- `Watch List`
  - current risks, active blockers, open incidents, compliance friction
- `Value & Commitments`
  - current value delivered, quantified proof points, open commitments, and services outcomes that reinforce renewal and expansion conversations
- `Competitive & Strategic`
  - strategic priorities, analytics posture, platform themes, competitive pressure, and source-backed expansion vectors
- `The Work`
  - current commitments, due dates, blockers, and recommended follow-through grounded in fresh source-backed evidence
- `The Record`
  - recent source rail, resolved incidents, older themes, delivered services work
- `Reports`
  - report entry points and summaries should inherit the same canonical account truth, not diverge from the dossier

### 3. Meeting briefing becomes evidence-first

Meeting detail / briefing should pull the strongest current account intelligence into a compact pre-meeting view:

- what matters most now
- who is on the call and what role they play
- likely customer concerns
- last 1-2 call themes
- recent incidents / pain points
- renewal state and blockers
- exact source references the user can chase in Glean

### 4. Verification UX

Every rendered fact or inference on account detail and meeting detail should be able to expose:

- source system
- source kind
- observed date
- exact reference title or ID
- a verification affordance such as "View source" or "Ask Glean for this record"

This should work for:

- opportunities
- tickets
- call recordings
- docs
- P2 posts

### 5. Timeliness in UI

Do not show stale content as current.

UI behaviors:

- active items are visible by default
- resolved or older items move into The Record
- current themes show recency badges
- support incidents show status and date
- older operational issues no longer dominate meeting prep unless they recur or remain unresolved

---

## Acceptance Criteria

1. Account hero vitals can auto-populate from DB-backed factual Glean evidence when the user has not manually set those fields.
2. A user-edited vital is never silently overwritten by a later source-backed refresh.
3. The account detail `Outlook` chapter can show renewal date, renewal likelihood, renewal opportunity stage, ARR, and procurement complexity with visible source attribution.
4. The account detail `The Room` chapter renders a relationship map that distinguishes champion, decision maker, procurement, security, and internal team roles.
5. The account detail `State of Play` and `Watch List` chapters render current support and operational context without over-weighting stale resolved incidents.
6. The account detail `The Record` chapter can surface resolved incidents, older themes, and completed services work as historical evidence.
7. The account detail `The Work` and `Reports` chapters inherit the same canonical fact layer and freshness rules as the rest of the dossier.
8. Meeting detail shows source-attributed sections for current concerns, recent incidents, stakeholder roles, and renewal blockers.
9. From both account detail and meeting detail, a user can identify the exact originating call, ticket, opportunity, or doc for surfaced content.

---

## Implementation Notes

- This is not a request for more CRUD. The page should remain editorial and dossier-like.
- Prefer chapter enrichment and stronger evidence rails over adding management widgets.
- This issue should preserve existing visual language while raising factual density and trust.
- The `VitalsStrip` remains an editable surface. The change is in how empty or non-overridden values are hydrated.
