## Summary

An `OpenAIProvider` that calls the OpenAI chat completions API with a user-supplied API key. Allows users who already pay for ChatGPT Plus or have an OpenAI API subscription to use their existing access for DailyOS entity enrichment instead of a Claude Code subscription.

Because Ollama's API is OpenAI-compatible, `OpenAIProvider` and `OllamaProvider` (I433) share the same underlying HTTP client â€” a single `CompletionApiClient` struct parameterized by base URL and auth method. The two providers are thin wrappers around that shared client.

## Acceptance Criteria

1. `OpenAIProvider` implements `IntelligenceProvider` (I432) in `src-tauri/src/intelligence/provider.rs`. It calls `https://api.openai.com/v1/chat/completions` with the user's API key in the `Authorization: Bearer <key>` header. The request uses the same OpenAI chat completions schema as `OllamaProvider`. The response is parsed from `choices[0].message.content`.

2. Settings â†’ AI Provider shows "OpenAI API" as a selectable option. Selecting it reveals:
   - An API key input field (masked). The placeholder reads "sk-...".
   - A "Test connection" button that makes a minimal call (`model: gpt-4o-mini`, single-token completion) to verify the key is valid before saving. Invalid or expired keys are rejected at this step with the raw API error message displayed.
   - A model selector for Synthesis tier (default: `gpt-4o`) and Extraction/Mechanical tiers (default: `gpt-4o-mini`).

3. API key storage: the key is stored in the macOS Keychain under the service name `com.dailyos.openai`. It is never written to the SQLite DB, the workspace config file, or any file on disk. Verify: after configuring an OpenAI key, `grep -rn "sk-" ~/.dailyos/` returns nothing (or the grep does not match any file containing the actual key value).

4. With OpenAI selected and a valid key, entity enrichment produces valid `entity_intel`. The `executive_assessment`, `key_risks`, and `open_threads` fields are non-null for a real account with sufficient signal history. Output quality is expected to be comparable to Claude for entity intelligence tasks â€” this is the user's responsibility to evaluate, but structural validity is the acceptance bar.

5. `cargo test` passes. `cargo clippy -- -D warnings` passes. No new warnings introduced.
