# I629 — Briefing expansion panel restoration

**Area:** Frontend / Daily Briefing
**Priority:** P1
**Version:** v1.0.2
**Effort:** Low (< 1 hour)
**Source:** CEO plan review + philosophy challenge (2026-03-21). Discovered during investigation of daily briefing current state.

---

## Problem

`BriefingMeetingCard.tsx` exports three sub-components for the meeting card expansion panel:

1. `KeyPeopleFlow` — "The Room" attendees grouped by side (Their/Our)
2. `PrepGrid` — 2-column grid: Discuss (first action/question), Watch (first risk), Wins (first win)
3. `MeetingActionChecklist` — "Before this meeting" action checklist (max 3 actions)

The expansion panel (lines 487-561) renders only:
- Calendar description or AI context brief
- `KeyPeopleFlow` (The Room)
- Bridge link ("Read full briefing →") + Collapse button

`PrepGrid` and `MeetingActionChecklist` are **exported but never rendered** in the expansion panel. This is a regression from a redesign pass — the components were built, exported, but the expansion panel inner content was never updated to include them.

The result: expanding a meeting on the daily briefing shows attendees and a link, but NOT the prep preview (discuss/watch/wins) or pre-meeting actions. The briefing's schedule section is thinner than intended.

## Solution

Reconnect `PrepGrid` and `MeetingActionChecklist` in the expansion panel inner content, after `KeyPeopleFlow` and before the bridge link.

## Implementation

In `src/components/dashboard/BriefingMeetingCard.tsx`, in the expansion panel `<div ref={innerRef}>` block (around line 492):

**After** the `KeyPeopleFlow` rendering (around line 537), **before** the bridge link div (around line 539), add:

```tsx
{/* Prep preview — discuss/watch/wins */}
<PrepGrid meeting={meeting} />

{/* Before this meeting — action checklist */}
<MeetingActionChecklist
  actions={meetingActions}
  completedIds={completedIds}
  onComplete={onComplete}
/>
```

The props are already available — `meetingActions`, `completedIds`, and `onComplete` are received by `BriefingMeetingCard` but currently unused (prefixed with `_` to suppress warnings: `_meetingActions`, `_onComplete`, `_completedIds`). Remove the `_` prefixes and pass them through.

Also remove `_onEntitiesChanged` prefix if it's similarly unused.

## Acceptance Criteria

1. Expand an upcoming meeting on the daily briefing that has prep content (risks, actions, discussion items). The expansion panel shows:
   - Calendar description or AI context
   - The Room (attendees grouped by side)
   - PrepGrid: Discuss (first item), Watch (first risk), Wins (first win) — each with colored dot
   - "Before this meeting" checklist (max 3 actions with completion circles)
   - "Read full briefing →" link + Collapse button
2. Meetings without prep content still expand cleanly — PrepGrid and MeetingActionChecklist return null when data is empty.
3. Meeting actions can be completed from the expansion panel (checkbox click).
4. "In progress" meetings auto-expand with all content visible.
5. No console errors. TypeScript clean.
