# I637 — Meeting-to-meeting continuity thread

**Area:** Backend / Intelligence + Frontend / Meeting Detail
**Priority:** P1
**Version:** v1.0.3
**Effort:** Small
**Depends on:** I634 (temporal transformation — the thread lives in Stage 4: Processed)
**Source:** CEO plan review (2026-03-21). "Context compounds" — each meeting should connect to the one before it.

---

## Problem

Meetings with the same entity happen in a sequence — weekly syncs, quarterly reviews, escalation follow-ups. Each meeting is an episode in an ongoing story. But the post-meeting record treats each meeting as isolated. There's no "here's what changed since the last time you met."

The pre-meeting prep already has a "Since Last Meeting" section with bulleted changes. But this data disappears in the post-meeting view — it's buried in the faded prep content. The post-meeting record should carry this forward as "The Thread" — connecting this meeting to the one before it.

## Solution

Include "The Thread" as a section in the Meeting Record (Stage 4 of I634), positioned between the headline and the prediction scorecard.

### Data Source

The `sinceLast` data is already computed for pre-meeting prep. It includes:
- Changes detected since the previous meeting with this entity
- Action items from the previous meeting and their status
- Signal changes (health, risk, engagement)

For the post-meeting record, extend this with:
- **Actions completed since last meeting** — query `actions` table for actions linked to this entity completed between `prev_meeting_date` and `this_meeting_date`
- **Risks resolved/escalated** — compare risks from previous meeting's transcript outcomes against current meeting's risks
- **Health score delta** — `entity_quality.health_score` change between meetings
- **New stakeholders** — people who attended this meeting but not the previous one

### Display

```
THE THREAD
Since your last meeting with Acme Corp on Mar 7...

• 2 actions completed: "Follow up on support ticket" ✓, "Send ROI summary" ✓
• 1 action still open: "Coordinate migration timeline" (14 days overdue)
• Health: 68 → 72 (+4) — email engagement improved
• New face: Sarah Chen (VP Engineering) — first time in a meeting with Acme
```

**First meeting with entity:** "First meeting with {entity_name}. No prior context."

**No linked entity:** Section omitted entirely.

## Acceptance Criteria

1. Processed meeting record shows "The Thread" section with items from since-last-meeting data.
2. Lists: actions completed, actions still open (with overdue flag), health delta, new attendees.
3. "First meeting with {entity}" shown when no previous meeting exists.
4. Section omitted if meeting has no linked entity.
5. Thread data appears in the meeting record markdown (I636).
6. Data reuses existing `sinceLast` computation from prep — doesn't recompute from scratch.
