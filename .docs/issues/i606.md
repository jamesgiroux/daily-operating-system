# I606 — Book of Business Rebuild

**Version:** v1.0.3 Phase 3
**Depends on:** I604 (schema, portfolio_assessment table)
**Type:** Feature
**Scope:** Backend (reports/book_of_business, reports/portfolio_enrichment, scheduler) + Frontend (BookOfBusinessPage)

## Problem

The BoB report is decommissioned due to:

1. **Schema mismatch** between the synthesis path and legacy monolithic path — different field names, different structures, dead code from multiple rewrites
2. **Rendering inconsistencies** — the frontend can't reliably render the output because the schema drifts between generations
3. **90-second timeout** too tight for large portfolios (other complex reports use 180s)
4. **Skeletal mechanical builders** — `build_expansion_accounts` hardcodes "Exploring" readiness, `build_year_end_outlook` sets `expected_churn = 0.0`
5. **Per-thread tokio runtimes** for Glean pre-fetch — expensive, functionally dubious
6. **No pre-computed portfolio intelligence** — the narrative sections (executive summary, leadership asks, themes) are generated on-demand from a single synthesis PTY call that can timeout or produce inconsistent results

## Design

### Architecture: Mechanical + Stored Narrative

The BoB splits cleanly into two types of content:

**Mechanical (Rust, instant, no AI):**
These sections read directly from per-entity `entity_assessment` + `entity_quality` + `accounts` tables and produce structured output via pure functions.

- Portfolio health distribution — count of green/yellow/red bands from `health_json`
- Account snapshot table — ARR, health, lifecycle, renewal date, meeting count, primary contact
- Risk accounts — at-risk/watch accounts sorted by ARR, with `risks_json[0]` as summary
- Expansion accounts — accounts with `expansion_signals` from `dimensions_json`, with evidence
- Renewal timeline — upcoming renewals from `dimensions_json.contract_context`
- Year-end outlook — arithmetic from total/at-risk ARR with `projected_churn` from portfolio_assessment
- Per-account deep dives — `executive_assessment` + `risks_json` + `open_commitments` per account, re-shaped for BoB cards

**Stored narrative (from `portfolio_assessment`, pre-generated):**

- `executive_summary` — cross-portfolio synthesis
- `leadership_asks_json` — cross-portfolio judgment: what the CSM needs from leadership
- `quarterly_focus_json` — retention/expansion/execution priorities
- `key_themes_json` — patterns across accounts with cited evidence

### Portfolio Synthesis Task

A new background task that runs when any entity's `enriched_at` changes and the aggregate `intel_hash` differs from `portfolio_assessment.intel_hash`.

**Input:** For each active customer account, extract:
- `executive_assessment` (first 200 chars)
- `health_json.band` + `.score`
- `risks_json` (top 2 by severity)
- `dimensions_json.expansion_signals` (top 1)
- `accounts.arr`
- `accounts.lifecycle`

**PTY call:** One call, `ModelTier::Synthesis`, 180-second timeout. Prompt includes aggregated per-account context (~50-100 tokens per account). Produces `executive_summary`, `leadership_asks`, `quarterly_focus`, `key_themes`, `projected_churn_narrative`.

**Storage:** Upsert to `portfolio_assessment`. The BoB report reads from this table — no PTY at report time.

**Glean integration:** If connected, pre-fetch risk_pulse and themes via Glean MCP (shared tokio runtime, not per-thread). Inject into synthesis prompt as supplementary context.

### Frontend

- Clean-sheet slide layout matching editorial design system
- Mechanical sections render instantly
- Narrative sections render from `portfolio_assessment`
- If `portfolio_assessment` is empty (first run), show mechanical sections with "Portfolio intelligence generating..." placeholder for narrative sections
- Progressive rendering: mechanical slides visible immediately, narrative slides populate when `portfolio_assessment` exists
- Feature flag gated: `book_of_business_enabled`

### Schema

Single coherent `BookOfBusinessContent` struct. No legacy/synthesis split.

```rust
pub struct BookOfBusinessContent {
    // Mechanical (instant)
    pub period_label: String,
    pub total_accounts: usize,
    pub total_arr: Option<f64>,
    pub at_risk_arr: Option<f64>,
    pub health_overview: PortfolioHealthOverview,
    pub account_snapshot: Vec<AccountSnapshotRow>,
    pub risk_accounts: Vec<RiskAccountRow>,
    pub expansion_accounts: Vec<ExpansionAccountRow>,
    pub renewal_timeline: Vec<RenewalRow>,
    pub year_end_outlook: YearEndOutlook,
    pub deep_dives: Vec<AccountDeepDive>,

    // Stored narrative (from portfolio_assessment)
    pub executive_summary: Option<String>,
    pub leadership_asks: Vec<LeadershipAsk>,
    pub quarterly_focus: Option<QuarterlyFocus>,
    pub key_themes: Vec<BookTheme>,
    pub projected_churn_narrative: Option<String>,
}
```

## Out of Scope

- Other report types (I605, I607, I608)
- Portfolio page (I492 — separate surface, can consume `portfolio_assessment` later)

## Acceptance Criteria

1. Zero PTY calls when opening the BoB report — all content from `portfolio_assessment` + mechanical assembly
2. Single coherent output schema — no legacy/synthesis split, no `#[allow(dead_code)]`
3. Mechanical sections render instantly from per-entity `entity_assessment`
4. Risk accounts show top risk from `risks_json` per account (not re-derived)
5. Expansion accounts show evidence from `dimensions_json.expansion_signals` (not hardcoded "Exploring")
6. Year-end outlook uses `projected_churn_narrative` from portfolio_assessment (not hardcoded 0.0)
7. Portfolio synthesis task runs in background when intel_hash changes
8. Portfolio synthesis has 180-second timeout
9. Graceful degradation: mechanical sections render even if portfolio_assessment is empty
10. Feature flag gated (`book_of_business_enabled`)
11. Glean pre-fetch uses shared runtime (not per-thread)
